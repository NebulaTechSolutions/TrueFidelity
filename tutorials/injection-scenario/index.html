    Injection Scenario Walkthrough¶

    This comprehensive tutorial demonstrates how to use TrueFidelity's signal injection capabilities to test ECU responses and validate system behavior. You'll learn how to inject CAN signals into a running system, monitor ECU reactions, and document test results. Signal injection is a powerful technique for validating control logic, testing edge cases, and reproducing field issues in a controlled environment.

    Time Estimate¶

    • System Setup: 10-15 minutes
    • Injection Configuration: 10-15 minutes
    • Test Execution: 15-20 minutes
    • Analysis and Documentation: 10-15 minutes
    • Total Time: 45-65 minutes

    Prerequisites¶

    Before starting this tutorial, ensure you have:

    • TrueFidelity Desktop application with a valid license
    • Docker Desktop running (or Mock Mode for simulation)
    • A configured system with CAN network and ECUs
    • DBC file with signal definitions
    • Basic understanding of CAN signals and ECU communication
    • Sample BLF file for reference data (optional)

    Safety Warning¶

    ⚠️ CRITICAL SAFETY INFORMATION ⚠️

    Signal injection can affect ECU behavior and system operation. Follow these safety guidelines:

    • Never inject signals into production vehicles or safety-critical systems
    • Only use injection on test benches or isolated systems
    • Ensure emergency stop procedures are in place
    • Verify ECU configurations allow test inputs
    • Monitor system behavior continuously during injection
    • Have a recovery plan if systems behave unexpectedly

    Learning Objectives¶

    By completing this tutorial, you will:

    • Understand signal injection concepts and capabilities
    • Learn to use the Signal Player & Injection panel
    • Configure and execute injection tests
    • Monitor ECU responses to injected signals
    • Analyze system behavior under test conditions
    • Document test procedures and results
    • Troubleshoot common injection issues

    Scenario Overview¶

    You're tasked with validating a new suspension controller's response to various road conditions. The controller should adjust damping based on wheel speed and acceleration signals. Your objectives are:

    1. Create a test system with the suspension controller
    2. Inject controlled signal patterns
    3. Verify the controller's damping adjustments
    4. Test edge cases and error conditions
    5. Document the validation results

    This scenario represents a typical hardware-in-the-loop (HIL) testing workflow used in automotive development.

    Part 1: System Preparation¶

    Step 1: Create the Test System¶

    Set up a system specifically for injection testing.

    1. Launch TrueFidelity Desktop:
    2. Start in System Design mode

    3. If you have an existing system, save any changes

    4. Create a new system:

    5. Click File → New System
    6. Name: SuspensionInjectionTest
    7. Description: "Suspension controller validation with signal injection"

    8. Click Create

    9. Add the CAN network:

    10. Drag a CAN Bus from Toolbox to canvas
    11. Select the CAN bus
    12. In Properties panel:
      • Name: Chassis-CAN
      • DBC File: Select suspension_control.dbc
      • Baud Rate: Auto-populated from DBC
      • Click Apply Changes

    Step 2: Add Test ECUs¶

    Configure ECUs for the injection test.

    1. Add the Suspension Controller ECU:
    2. Drag a Generic ECU from Toolbox
    3. Position above the CAN bus

    4. Configure in Properties:

      • Name: Suspension-Controller
      • Type: Linux Generic x86_64
      • Click Apply Changes

    5. Add the Signal Generator ECU:

    6. Drag another Generic ECU
    7. Position to the left

    8. Configure:

      • Name: Signal-Generator
      • Type: Linux Generic x86_64
      • Note: This ECU will handle injected signals

    9. Add the Monitor ECU:

    10. Drag a third Generic ECU
    11. Position to the right
    12. Configure:
      • Name: Monitor-ECU
      • Type: Linux Generic x86_64
      • Note: For observing system responses

    Step 3: Connect ECUs to Network¶

    Wire all ECUs to the CAN bus.

    1. Connect each ECU:
    2. Hover over each ECU
    3. Drag from connection handle to CAN bus

    4. Ensure all three are connected

    5. Verify in Project Explorer:

    6. Expand Networks → Chassis-CAN
    7. All three ECUs should be listed

    1. Save the system:
    2. Click File → Save
    3. Save as SuspensionInjectionTest.json

    Step 4: Build and Deploy¶

    Prepare the system for testing.

    1. Build the system:
    2. Click Build System in toolbar
    3. Monitor build console
    4. Wait for "Build completed successfully"

    1. Start the system:
    2. Click Start button
    3. Wait for all containers to run

    4. Status bar shows container count

    5. Verify readiness:

    6. All ECUs show as running
    7. No errors in console
    8. System ready for injection

    Part 2: Injection Setup¶

    Step 5: Switch to CAN Log Analysis Mode¶

    Configure the workspace for injection testing.

    1. Change mode:
    2. Click View → CAN Log Analysis

    3. Layout reconfigures for analysis

    4. Understand the layout:

    5. Signal Viewer (left) - Signal selection
    6. Network Traffic (center) - Live frames
    7. Signal Chart (right) - Real-time plotting
    8. Signal Player & Injection (bottom) - Injection controls

    1. Focus on injection panel:
    2. Locate Signal Player & Injection at bottom
    3. Note the Injection section with controls
    4. This is where you'll manage injection

    Step 6: Understanding Injection Controls¶

    Familiarize yourself with injection capabilities.

    1. Locate injection controls:
    2. In Signal Player & Injection panel
    3. Find the Injection section
    4. Available controls:
      • Start all - Begin injection
      • Pause all - Temporarily stop
      • Stop all - End injection

    1. Injection options:
    2. Signal selection for injection
    3. Value configuration
    4. Network target selection

    1. Important limitations:
    2. Injection sends predetermined signals
    3. No complex scheduling available
    4. All selected signals inject simultaneously
    5. Cannot create custom message templates

    Step 7: Select Signals for Injection¶

    Choose which signals to inject.

    1. Open Signal Viewer:
    2. Click on Signal Viewer panel (left)

    3. Shows all signals from DBC

    4. Identify target signals:

    Search for suspension-related signals: - Wheel_Speed_FL - Front left wheel speed - Wheel_Speed_FR - Front right wheel speed - Wheel_Speed_RL - Rear left wheel speed - Wheel_Speed_RR - Rear right wheel speed - Vehicle_Acceleration - Longitudinal acceleration - Steering_Angle - Steering wheel position

    1. Select injection signals:
    2. Check boxes next to wheel speed signals
    3. Select acceleration signal

    4. These will be available for injection

    5. Identify monitoring signals:

    Signals to observe (not inject): - Damping_Command_FL - Front left damper setting - Damping_Command_FR - Front right damper setting - Damping_Command_RL - Rear left damper setting - Damping_Command_RR - Rear right damper setting - Suspension_Mode - Active suspension mode - Diagnostic_Status - Controller diagnostics

    1. Select monitoring signals:
    2. Check these for charting
    3. Will show controller responses

    Part 3: Injection Execution¶

    Step 8: Configure Injection Values¶

    Set up the signal values to inject.

    1. Define test scenarios:

    Scenario 1 - Normal Driving: - All wheel speeds: 50 km/h - Acceleration: 0 m/s² - Expected: Normal damping

    Scenario 2 - Hard Braking: - All wheel speeds: Decreasing from 80 to 20 km/h - Acceleration: -8 m/s² - Expected: Increased front damping

    Scenario 3 - Cornering: - Different wheel speeds simulating turn - Lateral acceleration - Expected: Adjusted damping per wheel

    1. Prepare injection parameters:
    2. Start with Scenario 1 (steady-state)
    3. All wheels at constant speed

    4. Zero acceleration

    5. Configure signal values:

    6. Set each wheel speed to 50 km/h equivalent
    7. Set acceleration to 0
    8. Prepare for injection start

    Step 9: Start Signal Injection¶

    Begin the injection test.

    1. Final safety check:

    Before starting: - Confirm test bench setup only - Verify no production systems connected - Ensure monitoring is active - Have stop procedure ready

    1. Start injection:
    2. Click Start all in injection controls
    3. Injection begins immediately

    4. Signals transmit to the CAN bus

    5. Monitor Network Traffic:

    6. Watch Network Traffic panel
    7. Injected frames appear
    8. Note frame timing and frequency
    9. Verify CAN IDs match expectations

    1. Observe initial responses:
    2. Watch Signal Chart for changes
    3. Monitor damping commands
    4. Note suspension mode
    5. Check diagnostic status

    Step 10: Monitor System Response¶

    Analyze how the controller reacts.

    1. Watch Signal Chart:
    2. Damping commands should stabilize
    3. Values should match expected range
    4. No oscillations or instability

    1. Check controller behavior:
    2. Damping values appropriate for speed
    3. Suspension mode correct

    4. No diagnostic errors

    5. Document steady-state:

    6. Take screenshot of stable operation
    7. Record damping values

    8. Note any unexpected behavior

    9. Let run for validation:

    10. Continue injection for 30 seconds
    11. Ensure stability maintained
    12. Watch for any drift or changes

    Step 11: Test Dynamic Scenarios¶

    Simulate changing conditions.

    1. Transition to Scenario 2 (Braking):

    While injection is running: - Monitor current state - Prepare to change values - Note timestamp for transition

    1. Modify injection values:
    2. Pause injection briefly
    3. Adjust wheel speeds (simulate deceleration)
    4. Set negative acceleration

    5. Resume injection

    6. Observe braking response:

    7. Front damping should increase
    8. Rear damping may decrease
    9. Watch for anti-dive behavior

    10. Monitor mode changes

    11. Document dynamic response:

    12. Screenshot during transition
    13. Record response time
    14. Note damping changes
    15. Check for proper behavior

    Step 12: Test Edge Cases¶

    Validate error handling and limits.

    1. Test invalid inputs:

    Scenario 4 - Sensor Failure: - Set one wheel speed to 0 - Others at normal speed - Expected: Diagnostic flag, fallback mode

    1. Execute edge case test:
    2. Stop current injection
    3. Configure abnormal values
    4. Start injection again

    5. Monitor diagnostic response

    6. Observe error handling:

    7. Controller should detect issue
    8. Diagnostic status should change
    9. Fallback strategy should engage

    10. System should remain stable

    11. Test recovery:

    12. Return to normal values
    13. Verify system recovers
    14. Check diagnostic clears
    15. Confirm normal operation resumes

    Step 13: Stop Injection¶

    Complete the test sequence.

    1. Prepare to stop:
    2. Ensure all data captured
    3. Take final screenshots

    4. Note ending timestamp

    5. Stop injection:

    6. Click Stop all button
    7. Injection ceases immediately

    8. CAN traffic returns to idle

    9. Verify system state:

    10. Controller returns to standby
    11. No residual effects
    12. Diagnostics clear
    13. System stable

    Save evidence for reporting.

    1. Export Network Traffic:

    3. Click Export in Network Traffic
    4. Choose CSV format
    5. Name: injection_test_traffic.csv

    6. Includes all injected frames

    7. Export Signal Data:

    8. Ensure all test signals selected

    10. Name: injection_test_signals.csv

    11. Contains response measurements

    12. Export visualizations:

    13. Right-click Signal Chart
    14. Export as PNG
    15. Save key moments:
      • Steady-state operation

    Step 16: Create Test Report¶

    Document the injection test.

    1. Compile test documentation:

    ``` Suspension Controller Injection Test Report Date: [Current Date] Test Engineer: [Your Name] System: SuspensionInjectionTest.json

    Test Objective: Validate suspension controller response to various driving conditions using signal injection.

    Test Setup: - Controller: Suspension-Controller ECU - Network: Chassis-CAN (500 kbps) - DBC: suspension_control.dbc - Injection method: Signal Player & Injection

    Test Scenarios: 1. Normal Driving (50 km/h steady) - Result: PASS - Damping: 50% all wheels - Response: <100ms

    1. Hard Braking (80→20 km/h, -8 m/s²)

      • Result: PASS
      • Front damping: Increased to 75%
      • Rear damping: Decreased to 30%
      • Anti-dive active

    2. Cornering (differential wheel speeds)

      • Result: PASS
      • Outside wheels: 65% damping
      • Inside wheels: 45% damping
      • Roll control active

    3. Sensor Failure (FL wheel = 0)

      • Result: PASS
      • Diagnostic flag set
      • Fallback mode engaged
      • Recovery successful

    Conclusions: - Controller meets all performance requirements - Response times within specification - Error handling robust - Ready for vehicle integration testing

    Attachments: - injection_test_traffic.csv - injection_test_signals.csv - response_charts.png ```

    1. Include supporting evidence:
    2. CSV exports
    3. Chart screenshots
    4. System configuration

    5. Test parameters

    6. Save and distribute:

    7. Store with project files
    8. Share with development team
    9. Archive for compliance

    Troubleshooting Guide¶

    Common Injection Issues¶

    Injection Won't Start¶

    Symptom: Start all button clicked but no injection

    Solutions:

    1. Verify system is running
    2. Check ECUs are connected
    3. Ensure signals are selected
    4. Verify network configuration
    5. Check for error messages

    No ECU Response¶

    Symptom: Injection active but no reaction

    Solutions:

    1. Verify ECU receives messages
    2. Check signal scaling in DBC
    3. Ensure ECU expects test inputs
    4. Verify ECU is programmed
    5. Check diagnostic mode

    Unexpected Behavior¶

    Symptom: ECU responds incorrectly

    Solutions:

    1. Verify injection values are correct
    2. Check DBC signal definitions
    3. Review ECU specifications
    4. Test with known good values
    5. Check ECU firmware version

    System Instability¶

    Symptom: Oscillations or erratic behavior

    Solutions:

    1. Stop injection immediately
    2. Review injection rate
    3. Check for feedback loops
    4. Verify value ranges
    5. Reduce injection frequency

    Network Overload¶

    Symptom: Missing frames or delays

    Solutions:

    1. Reduce number of injected signals
    2. Lower injection frequency
    3. Check bus utilization
    4. Optimize message timing
    5. Use filtering to reduce load

    Safety Best Practices¶

    Pre-Test Checklist¶

    Before any injection test:

    • ✓ Test bench only (no production systems)
    • ✓ Emergency stop procedure ready
    • ✓ Team notified of test
    • ✓ Baseline behavior documented
    • ✓ Recovery plan prepared
    • ✓ Monitoring active
    • ✓ Data recording enabled

    During Test¶

    While injection is active:

    • Monitor continuously
    • Watch for unexpected behavior
    • Be ready to stop immediately
    • Document any anomalies
    • Keep test duration reasonable
    • Avoid extreme values initially

    Post-Test¶

    After injection:

    • Verify system returns to normal
    • Check for residual effects
    • Clear any diagnostic codes
    • Document all findings
    • Archive test data
    • Review for improvements
