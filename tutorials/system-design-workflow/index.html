    System Design Workflow Tutorial¶

    This tutorial walks you through designing and deploying a complete ECU system in TrueFidelity, from initial setup to running and monitoring containers.

    Overview¶

    Goal: Create a multi-ECU system with network connections, configure build settings, deploy containers, and monitor their execution.

    Time estimate: 15-20 minutes

    Prerequisites: - TrueFidelity installed and activated - Docker Desktop running - ECU firmware binaries available (or use sample builds)

    Step 1: Create a New System¶

    1. Launch TrueFidelity
    2. Go to File → New (or press Ctrl/Cmd + N)

    1. In the dialog, enter:
    2. System name: tutorial-gateway-system
    3. Save location: Choose your projects folder

    1. Click Create

    The System Canvas opens with an empty workspace. The Toolbox panel should be visible on the left.

    Note

    If the Toolbox isn't visible, open it via View → Panels → Toolbox or press the toolbar button.

    Step 2: Add ECUs to the Canvas¶

    1. In the Toolbox, locate the ECUs category

    1. Find a Gateway ECU component
    2. Drag it onto the canvas

    1. Repeat to add two more ECUs:
    2. A BCM (Body Control Module)
    3. A PCM (Powertrain Control Module)

    Your canvas should now show three ECU nodes. Position them with some space between them.

    Tip

    Use the canvas zoom controls (mouse wheel or toolbar) to adjust the view. Right-click and drag to pan.

    Step 3: Configure ECU Properties¶

    1. Click on the Gateway ECU to select it

    1. The Properties Panel (right side) shows ECU configuration
    2. Edit the following properties:
    3. Name: gateway-ecu (or leave default)
    4. Binary path: Browse to your ECU binary file, or leave empty for now
    5. Build target: Select from dropdown (e.g., s32g274a-ubuntu)

    1. Repeat for the BCM and PCM ECUs, giving them distinct names

    Note

    Binary paths can be set later. For this tutorial, you can proceed without binaries to see the system structure.

    Step 4: Create Network Connections¶

    1. Click and hold on the Gateway ECU node
    2. Drag a connection line to the BCM ECU

    1. Release to create a network edge
    2. Repeat to connect Gateway to PCM

    You should now have two network connections. The edges represent CAN bus or Ethernet links.

    Step 5: Configure Network Properties¶

    1. Click on a network edge (connection line) to select it
    2. In the Properties Panel, configure:
    3. Network type: Choose CAN or Ethernet
    4. Channel: can0 (or appropriate for your setup)
    5. Baud rate: 500000 (for CAN)
    6. Repeat for the other network edge

    Step 6: Save the System¶

    1. Press Ctrl/Cmd + S or go to File → Save
    2. Confirm the save location
    3. The system configuration is saved as a .tfsystem file

    Your system design is now complete and saved.

    Step 7: Build Firmware (Optional)¶

    If you have firmware source code:

    1. Select an ECU in the canvas
    2. In Properties, click Build Configuration
    3. Select build target and compiler options
    4. Click Build to compile
    5. Monitor progress in the Build Console panel

    For this tutorial, you can skip building if you don't have source code ready.

    Step 8: Deploy and Start ECUs¶

    1. In the ECU Status panel, you should see your three ECUs listed

    1. Right-click on the Gateway ECU
    2. Select Start from the context menu

    1. Wait for the status to change to "Running" (green indicator)

    1. Repeat for BCM and PCM ECUs

    Tip

    You can also use the toolbar buttons or keyboard shortcuts to start/stop ECUs. Select multiple ECUs (Ctrl/Cmd + Click) and start them all at once.

    Step 9: Monitor Running System¶

    View ECU Status¶

    1. Open the ECU Status panel (if not already visible)
    2. Verify all three ECUs show "Running" status
    3. Check resource usage columns (CPU%, Memory)

    View Console Logs¶

    1. Open the Console panel
    2. You should see log messages from all ECUs

    1. Use the filter bar to:
    2. Filter by ECU name (e.g., type gateway)
    3. Filter by log level (toggle info/warn/error/debug)
    4. Search for specific text

    Monitor Resources¶

    1. Open the Resource Monitor panel
    2. Watch real-time CPU and memory graphs
    3. Each ECU has a colored line on the graph
    4. Adjust the time window using the dropdown (1min, 5min, etc.)

    View Network Traffic¶

    1. Open the Network Traffic panel
    2. See live CAN messages flowing between ECUs
    3. Filter by message ID, channel, or source ECU
    4. Click any message to see detailed payload

    Access ECU Terminals¶

    1. Open the Terminal panel
    2. You'll see tabs for each running ECU

    1. Click the gateway-ecu tab
    2. You now have shell access to that ECU's operating system
    3. Try running commands like ls, ps, or cat /proc/meminfo

    Warning

    Terminal access depends on ECU configuration. Some ECUs may require special setup or have restricted access.

    Step 10: Make Changes to Running System¶

    Stop an ECU¶

    1. In ECU Status panel, right-click on PCM
    2. Select Stop
    3. Watch the status change to "Stopped"
    4. Console logs will show shutdown messages

    Restart an ECU¶

    1. Right-click the stopped ECU
    2. Select Start
    3. ECU restarts and logs appear in Console

    Modify Network Configuration¶

    1. Stop all ECUs (select all, right-click, Stop)
    2. In System Canvas, select a network edge
    3. Change properties (e.g., baud rate)
    4. Save the system
    5. Restart ECUs to apply changes

    Step 11: Export System Configuration¶

    1. Right-click on the System Canvas
    2. Select Export Configuration
    3. Choose format:
    4. JSON: For programmatic access
    5. YAML: For version control and infrastructure-as-code
    6. Save the file

    You can now share this configuration with teammates or use it in automated deployments.

    Step 12: Cleanup¶

    1. Stop all ECUs (select all in ECU Status, right-click, Stop)
    2. Close the system (File → Close)
    3. Optionally delete the system file if this was just a tutorial

    Troubleshooting¶

    ECUs won't start¶

    • Verify Docker Desktop is running
    • Check Console panel for error messages
    • Ensure binary paths are correct if specified
    • Try starting one ECU at a time to isolate issues

    Terminal connection fails¶

    • Verify ECU is running (green status in ECU Status panel)
    • Check that terminal service is enabled in ECU configuration
    • Review Console logs for connection errors
    • Some ECU images may not have terminal support enabled

    Network traffic not appearing¶

    • Ensure network edges are properly configured
    • Verify ECU firmware is sending CAN messages
    • Check Network Traffic panel filters aren't hiding messages
    • Confirm network type matches ECU configuration
