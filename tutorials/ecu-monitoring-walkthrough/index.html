    ECU Monitoring Walkthrough¶

    This tutorial focuses on monitoring and debugging running ECU systems in TrueFidelity. You'll learn to use monitoring panels effectively to track performance, diagnose issues, and gather diagnostic data.

    Overview¶

    Goal: Monitor a running ECU system, use filtering and analysis tools, access terminals for debugging, and export monitoring data.

    Time estimate: 10-15 minutes

    Prerequisites: - A running ECU system (or follow the System Design Workflow tutorial first) - TrueFidelity installed and activated

    Step 1: Set Up Monitoring Layout¶

    Before monitoring, organize your panels for optimal visibility:

    1. Open View → Reset Layout to start with defaults
    2. Open the following panels:
    3. ECU Status (Ctrl/Cmd + 1)
    4. Console (Ctrl/Cmd + 2)
    5. Resource Monitor (Ctrl/Cmd + 3)
    6. Network Traffic (if monitoring CAN)

    7. Terminal (Ctrl/Cmd + 4)

    8. Arrange panels to your preference:

    9. Drag panel tabs to reposition
    10. Resize panels by dragging edges

    11. Split views by dragging to edge docking zones

    12. Save this layout: View → Save Layout As... → name it "Monitoring"

    Tip

    Create different layouts for different monitoring scenarios (e.g., "Performance Monitoring", "Debugging Layout").

    Step 2: Monitor ECU Status¶

    The ECU Status panel provides an overview of all ECUs in your system.

    Understanding Status Indicators¶

    1. Open ECU Status panel
    2. Observe status colors:
    3. Green: Running normally
    4. Yellow: Starting up
    5. Red: Error state
    6. Gray: Stopped

    Quick Actions¶

    1. Right-click any ECU in the status table
    2. Context menu options:
    3. Start: Launch the ECU container
    4. Stop: Gracefully shutdown
    5. Restart: Stop then start
    6. View Logs: Open ECU-specific console view
    7. Terminal: Open terminal tab for this ECU
    8. Properties: View detailed configuration

    Resource Summary¶

    1. Hover over CPU% or Memory cells
    2. Tooltip shows detailed metrics
    3. Sort columns by clicking headers to identify resource-heavy ECUs

    Step 3: Analyze Console Logs¶

    The Console panel aggregates logs from all ECUs.

    Filtering Logs¶

    1. Open Console panel
    2. In the filter bar, type an ECU name (e.g., gateway)

    1. Only logs from that ECU are shown
    2. Clear filter to see all logs again

    Filter by Log Level¶

    1. Use log level toggles (Info, Warn, Error, Debug)
    2. Uncheck Info to focus on warnings and errors
    3. Check only Error to see critical issues

    Search Logs¶

    1. Click in the search box
    2. Type a keyword (e.g., timeout, error, CAN)
    3. Matching lines are highlighted
    4. Use arrow keys to navigate between matches

    Export Logs¶

    1. Apply desired filters
    2. Click Export button (or right-click → Export)
    3. Choose format:
    4. Text file: Plain text with timestamps
    5. CSV: For analysis in spreadsheet tools
    6. Save filtered logs for later analysis

    Tip

    Export logs periodically during long test runs to avoid losing data if the app crashes.

    Step 4: Track Resource Usage¶

    The Resource Monitor shows real-time CPU and memory trends.

    Reading the Graphs¶

    1. Open Resource Monitor panel

    1. Each ECU has a colored line:
    2. CPU usage as percentage
    3. Memory usage in megabytes
    4. X-axis shows time, Y-axis shows usage percentage/amount

    Adjust Time Range¶

    1. Use dropdown to select:
    2. 1 minute: Short-term spikes
    3. 5 minutes: Recent activity
    4. 15 minutes: Medium-term trends
    5. 1 hour: Long-term patterns

    Zoom into Specific Periods¶

    1. Click and drag on the graph to select a time range
    2. Graph zooms to that range
    3. Click Reset Zoom to return to full view

    Identify Issues¶

    • CPU spikes: Sudden increases may indicate processing bottlenecks
    • Memory leaks: Gradually increasing memory suggests memory leaks
    • Resource contention: Multiple ECUs high usage simultaneously

    Export Metrics¶

    1. Click Export button
    2. Choose format:
    3. CSV: For Excel or Python analysis
    4. PNG: For reports and presentations
    5. Exported data includes all visible time range

    Step 5: Monitor Network Traffic¶

    The Network Traffic panel shows CAN messages in real-time.

    View Live Messages¶

    1. Open Network Traffic panel
    2. Messages appear as they're transmitted
    3. Columns show: Timestamp, ID, DLC, Payload, Channel, Source

    Filter Messages¶

    1. In filter bar, enter:
    2. Message ID: 0x123 or 291 (decimal)
    3. Channel: can0, can1
    4. Source ECU: ECU name
    5. Only matching messages are shown

    Analyze Message Rates¶

    1. Watch the Rate column (messages/second)
    2. High rates may indicate excessive traffic
    3. Zero rates might indicate a broken connection

    View Signal Decoding¶

    1. Load a DBC file: Project → Import Definitions
    2. Signals are decoded and displayed below raw payload
    3. Click message to see decoded signal values

    Capture Traffic to Log¶

    1. Click Start Capture
    2. Traffic is recorded to memory
    3. Click Stop Capture
    4. Click Export to save as:
    5. BLF: Vector format
    6. MDF4: MDF format
    7. CSV: Spreadsheet format

    Step 6: Use Terminal Access¶

    The Terminal panel provides shell access to ECU operating systems.

    Connect to ECU¶

    1. Open Terminal panel
    2. Tabs show one terminal per ECU
    3. Click tab for the ECU you want to access
    4. Connection status indicator shows:
    5. Green: Connected
    6. Yellow: Connecting
    7. Red: Connection failed

    Run Diagnostic Commands¶

    Once connected, try these commands:

    # Check CPU usage
top

# View memory info
cat /proc/meminfo

# List processes
ps aux

# Check network interfaces
ifconfig

# View recent log messages
dmesg | tail -20

# Check disk space
df -h

    Search Terminal Output¶

    1. Press Ctrl+F (Cmd+F on macOS)
    2. Search box appears
    3. Type search term
    4. Matching text is highlighted
    5. Use arrow keys to navigate matches

    Terminal Actions¶

    • Clear: Button clears terminal screen (history preserved)
    • Copy: Select text with mouse, use Ctrl/Cmd+C
    • Reconnect: If connection drops, click reconnect button

    Warning

    Some commands may require root privileges. Be cautious with system modifications.

    Step 7: Troubleshoot Issues¶

    ECU Not Starting¶

    1. Check ECU Status panel for error state
    2. Open Console panel, filter to that ECU
    3. Look for error messages
    4. Check resource limits in Properties panel

    High Resource Usage¶

    1. Open Resource Monitor
    2. Identify which ECU is using resources
    3. Open Terminal for that ECU
    4. Run top to see process breakdown
    5. Check for runaway processes or memory leaks

    Network Issues¶

    1. Check Network Traffic panel for messages
    2. If no messages, verify:
    3. Network edges are connected in System Canvas
    4. ECU firmware is sending messages
    5. Filters aren't hiding all traffic
    6. Check Console logs for network errors

    Terminal Connection Problems¶

    1. Verify ECU is running (green in ECU Status)
    2. Check Console for terminal service errors
    3. Try reconnecting from Terminal panel
    4. Some ECU images may not support terminals

    Step 8: Export Monitoring Data¶

    Create Monitoring Report¶

    1. Export from each panel:
    2. Console: Text/CSV logs
    3. Resource Monitor: CSV metrics or PNG graphs
    4. Network Traffic: BLF/MDF4 capture
    5. Combine exports into a report document

    Automated Data Collection¶

    For long test runs:

    1. Set Resource Monitor time range to capture full duration
    2. Periodically export Console logs
    3. Capture Network Traffic at key moments
    4. Save Terminal command outputs

    Best Practices¶

    Organize Panels by Task¶

    • Development: Canvas, Properties, Build Console
    • Monitoring: ECU Status, Console, Resource Monitor
    • Debugging: Terminal, Process Monitor, Console
    • Analysis: Network Traffic, Signal Viewer, Charts

    Use Filters Effectively¶

    • Don't filter too aggressively—you might miss important events
    • Save filter presets for common scenarios
    • Combine multiple filters for precise analysis

    Export Regularly¶

    • Don't wait until the end to export data
    • Periodic exports prevent data loss
    • Name exports with timestamps for organization

    Monitor Baselines¶

    • Establish normal resource usage baselines
    • Compare current metrics to baselines to spot anomalies
    • Track trends over time
