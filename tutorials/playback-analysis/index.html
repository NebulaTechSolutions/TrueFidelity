    Playback Analysis Walkthrough¶

    This comprehensive tutorial focuses on analyzing CAN bus data through log playback, signal decoding, and pattern identification. You'll learn how to load captured logs, decode signals using DBC files, identify anomalies through visualization, and export findings for further analysis. This walkthrough represents a typical debugging workflow that automotive engineers use to diagnose system issues.

    Time Estimate¶

    • Initial Setup: 5-10 minutes
    • Playback Configuration: 10-15 minutes
    • Signal Analysis: 15-20 minutes
    • Data Export: 5-10 minutes
    • Total Time: 35-55 minutes

    Prerequisites¶

    Before starting this tutorial, ensure you have:

    • TrueFidelity Desktop application running
    • A system with at least one CAN network configured
    • Sample BLF log files for playback
    • DBC file with signal definitions
    • Basic understanding of CAN signals and data analysis

    Learning Objectives¶

    By completing this tutorial, you will:

    • Master the Signal Player panel for BLF playback
    • Understand signal decoding with DBC files
    • Learn to identify patterns in time-series data
    • Use filtering to focus on relevant signals
    • Export data for external analysis
    • Troubleshoot common playback issues

    Scenario Overview¶

    You're investigating cooling system behavior during a thermal endurance test. The test engineers report intermittent cooling fan issues that may be causing temperature spikes. Your task is to:

    1. Load and analyze the captured CAN log
    2. Identify cooling system signals
    3. Find anomalies in fan operation
    4. Quantify temperature variations
    5. Document findings for the team

    The log contains 15 minutes of data from a vehicle test, including engine parameters, cooling system status, and diagnostic messages.

    Part 1: System Preparation¶

    Step 1: Create or Load a System¶

    Before analyzing logs, you need a configured system with a CAN network.

    1. Launch TrueFidelity Desktop:
    2. The application opens in System Design mode

    3. Check if you have an existing system or need to create one

    4. Create a new system (if needed):

    5. Click File → New System
    6. Name it CoolingSystemAnalysis

    7. Click Create

    8. Add a CAN network:

    9. From the Toolbox, drag a CAN Bus to the canvas
    10. Select the CAN bus
    11. In Properties panel:
      • Name: Vehicle-CAN
      • DBC File: Browse and select powertrain.dbc
      • Click Apply Changes

    1. Add analysis ECUs (optional but recommended):
    2. Drag a Generic ECU from Toolbox
    3. Name it DataLogger-ECU
    4. Connect it to the CAN bus
    5. This ECU will receive the played back messages

    1. Save the system:
    2. Click File → Save
    3. Save as CoolingSystemAnalysis.json

    Step 2: Build and Start the System¶

    Deploy the system to enable playback functionality.

    1. Build the system:
    2. Click Build System in the toolbar
    3. Wait for "Build completed successfully"
    4. If using Mock Mode, build completes immediately

    1. Start the system:
    2. Click Start button (green play icon)
    3. Status bar shows "Starting system..."

    4. Wait for containers to start (10-30 seconds)

    5. Verify system is running:

    6. Status bar shows container count
    7. No error messages in console
    8. Ready for playback operations

    Part 2: CAN Log Analysis Setup¶

    Step 3: Switch to CAN Log Analysis Mode¶

    Configure the workspace for signal analysis.

    1. Change to analysis mode:
    2. Click View → CAN Log Analysis

    3. The layout reconfigures automatically

    4. Understand the analysis layout:

    5. Signal Viewer (left) - Hierarchical signal tree
    6. Network Traffic (center) - Live CAN frames
    7. Signal Chart (right) - Time-series visualization
    8. Signal Player & Injection (bottom) - Playback controls

    1. Adjust panel sizes if needed:
    2. Drag panel borders to resize
    3. Maximize Signal Chart for better visibility
    4. Ensure Signal Player is fully visible

    Step 4: Load the BLF Log File¶

    Import the captured data for analysis.

    1. Open Signal Player panel:
    2. Located at the bottom of the screen

    3. If not visible: View → Panels → Signal Player

    4. Load the BLF file:

    5. Click Select BLF button
    6. Navigate to your log file: cooling_endurance_test.blf
    7. Click Open

    1. Inspect the loaded data:
    2. Click Inspect bundle to view details:
      • File size and duration
      • Number of CAN frames
      • Detected channels
      • Average data rate

    3. Verify the log loaded correctly

    4. Check DBC association:

    5. The Networks section shows Vehicle-CAN
    6. Confirms DBC file is linked
    7. Signals will be decoded during playback

    Step 5: Configure Playback Settings¶

    Prepare for optimal playback performance.

    1. Set initial playback speed:
    2. Default is 1x (real-time)
    3. Can adjust during playback

    4. Start with 1x for initial review

    5. Verify system readiness:

    6. System must be running (toolbar shows Stop button)
    7. Signal Player shows Play button enabled

    8. Network Traffic panel is clear and ready

    9. Prepare monitoring panels:

    10. Position Network Traffic for visibility
    11. Clear any existing data with Clear button
    12. Ready Signal Viewer for navigation

    Part 3: Signal Identification and Selection¶

    Step 6: Locate Cooling System Signals¶

    Find the relevant signals for analysis.

    1. Open Signal Viewer:
    2. Click on the Signal Viewer panel (left)

    3. Shows all signals from the DBC file

    4. Search for cooling signals:

    5. Use the search box at top
    6. Enter "cool" to find cooling-related signals

    7. Results filter as you type

    8. Identify key signals:

    Look for these critical signals: - Engine_Coolant_Temp - Temperature reading - Cooling_Fan_Command - Fan control signal - Cooling_Fan_Speed - Actual fan RPM - Cooling_Fan_Status - Operational status

    1. Expand signal details:
    2. Click arrows next to CAN frames
    3. View signal properties:
      • Signal name and units
      • Min/max values
      • Scaling factors
      • Current value (during playback)

    Step 7: Select Signals for Monitoring¶

    Choose signals to chart and analyze.

    1. Select primary signals:
    2. Check the box next to Engine_Coolant_Temp
    3. Select Cooling_Fan_Command
    4. Select Cooling_Fan_Speed

    5. These will appear in the Signal Chart

    6. Add supporting signals:

    7. Vehicle_Speed - For context
    8. Engine_RPM - Load correlation

    9. Ambient_Temp - Environmental factor

    10. Clear search to see all signals:

    11. Remove search text
    12. Full signal tree returns
    13. Selected signals remain checked

    Part 4: Playback and Analysis¶

    Step 8: Start Playback¶

    Begin replaying the captured data.

    1. Start playback:
    2. In Signal Player, click Play button
    3. Playback begins from the start

    4. Progress bar shows current position

    5. Monitor Network Traffic:

    6. CAN frames appear in real-time
    7. Observe frame rate and timing
    8. Each frame shows:
      • Timestamp
      • CAN ID (hex)
      • Data bytes
      • Frame counter

    1. Watch Signal Viewer:
    2. Selected signals show live values
    3. Values update as frames arrive

    4. Color coding indicates changes

    5. Observe Signal Chart:

    6. Lines plot in real-time
    7. Each signal has unique color
    8. Time axis scrolls automatically

    Step 9: Identify Anomalies¶

    Look for unusual patterns in the data.

    1. Monitor temperature behavior:
    2. Watch Engine_Coolant_Temp line
    3. Normal range: 85-95°C

    4. Look for sudden spikes or drops

    5. Correlate fan operation:

    6. Compare Cooling_Fan_Command with Cooling_Fan_Speed
    7. They should align closely

    8. Delays indicate potential issues

    9. Identify problem areas:

    When you spot an anomaly: - Click Pause in Signal Player - Note the timestamp - Observe signal values - Look for patterns: - Fan command high but speed low - Temperature rising despite fan command - Intermittent fan speed drops

    1. Document findings:
    2. Take screenshot of Signal Chart
    3. Record timestamp ranges
    4. Note signal values at anomaly

    Step 10: Detailed Investigation¶

    Zoom into problem areas for closer analysis.

    1. Navigate to specific timestamp:
    2. Use Signal Player progress bar
    3. Click and drag to scrub through log

    4. Or enter specific time if known

    5. Adjust playback speed:

    6. Slow to 0.5x for detailed observation
    7. Speed up to 2x for quick scanning

    8. Pause frequently to examine details

    9. Focus on anomaly window:

    10. Identify start of anomaly
    11. Play through event slowly
    12. Identify end of anomaly

    13. Note total duration

    14. Chart analysis:

    In Signal Chart panel: - Time Window: Adjust to 30 seconds - Auto-scale: Enable for best fit - Grid: Enable for measurement - Zoom with mouse wheel on time axis

    Step 11: Filter Network Traffic¶

    Focus on specific messages during analysis.

    1. Apply CAN ID filter:
    2. In Network Traffic panel
    3. Enter CAN ID in filter field
    4. Example: 0x200 for cooling messages

    5. Click Apply Filter

    6. Filter by data pattern:

    7. Look for specific data values
    8. Useful for finding state changes

    9. Can combine with ID filter

    10. Clear filters when done:

    11. Click Clear Filter
    12. Returns to full traffic view
    13. Helps see context again

    Part 5: Quantitative Analysis¶

    Step 12: Measure Temperature Variations¶

    Quantify the severity of cooling issues.

    1. Identify temperature spikes:
    2. Pause at each spike peak
    3. Record maximum temperature
    4. Note baseline temperature

    5. Calculate delta (e.g., +15°C)

    6. Measure event duration:

    7. Note timestamp at spike start
    8. Note timestamp at recovery
    9. Calculate duration in seconds

    10. Document recovery time

    11. Create event summary:

    Example findings: ``` Event 1: - Time: 00:03:45 - Peak temp: 108°C (baseline: 92°C) - Duration: 35 seconds - Fan speed during event: 0 RPM

    Event 2: - Time: 00:07:22 - Peak temp: 105°C - Duration: 28 seconds - Fan speed: Intermittent ```

    Step 13: Pattern Analysis¶

    Identify recurring issues or triggers.

    1. Look for patterns:
    2. Do spikes occur regularly?
    3. Any correlation with vehicle speed?
    4. Related to engine load (RPM)?

    5. Environmental factors?

    6. Check fan response:

    7. Measure delay between command and speed
    8. Normal response: <500ms
    9. Delayed response indicates problem

    10. No response indicates failure

    11. System behavior:

    12. Does issue self-recover?
    13. Any diagnostic codes appear?
    14. Other systems affected?

    Part 6: Data Export¶

    Step 14: Export Network Traffic¶

    Save raw CAN data for detailed analysis.

    1. Stop playback:
    2. Click Stop in Signal Player

    3. Ensures complete data in buffer

    4. Export from Network Traffic:

    5. Click Export button
    6. Choose format:
      • CSV for spreadsheet analysis
      • BLF for replay in other tools
    7. Name: cooling_analysis_traffic.csv

    8. Click Save

    9. Export contents:

    10. All frames from playback session
    11. Includes timestamps and raw data
    12. Can be filtered before export

    Step 15: Export Signal Data¶

    Save decoded signal values.

    1. Prepare for export:
    2. Ensure relevant signals selected
    3. Playback includes full anomaly

    4. Chart shows complete event

    5. Export signal values:

    6. Focus on critical signals
    7. Include time range of interest
    8. Decoded values with units

    9. Save as: cooling_signals_decoded.csv

    10. Export includes:

    11. Timestamp column
    12. Selected signal columns
    13. Proper scaling applied
    14. Engineering units included

    Step 16: Export Visualizations¶

    Capture charts for documentation.

    1. Configure Signal Chart:
    2. Adjust time window to show anomaly
    3. Ensure all signals visible
    4. Enable grid for clarity

    5. Show legend with signal names

    6. Export chart:

    7. Right-click on Signal Chart
    8. Select Export Chart
    9. Choose format (PNG recommended)

    10. Name: temperature_spike_chart.png

    11. Take additional screenshots:

    12. Network Traffic during anomaly
    13. Signal Viewer with values
    14. Overall workspace layout
    15. Save for report inclusion

    Part 7: Documentation¶

    Step 17: Create Analysis Report¶

    Document findings for engineering team.

    1. Compile findings document:

    Create a structured report: ``` Cooling System Analysis Report Date: [Current Date] Analyst: [Your Name] Log File: cooling_endurance_test.blf Duration: 15 minutes

    Summary: Identified 3 cooling fan dropout events causing temperature spikes above 105°C.

    Details: 1. Event 1 - 00:03:45 - Temperature rise: 92°C to 108°C - Duration: 35 seconds - Fan status: Complete failure

    1. Event 2 - 00:07:22

      • Temperature rise: 90°C to 105°C
      • Duration: 28 seconds
      • Fan status: Intermittent operation

    2. Event 3 - 00:11:15

      • Temperature rise: 91°C to 106°C
      • Duration: 31 seconds
      • Fan status: No response to command

    Analysis: - Fan fails to respond to command signal - Failure occurs under high load conditions - Recovery is automatic after temperature peak - Pattern suggests electrical/relay issue

    Recommendations: - Check fan relay connections - Verify power supply under load - Review fan control logic - Consider redundant cooling strategy ```

    1. Attach supporting data:
    2. Exported CSV files
    3. Chart screenshots

    4. Network traffic logs

    5. Save documentation:

    6. Store with project files
    7. Include in version control
    8. Share with team

    Troubleshooting Guide¶

    Common Playback Issues¶

    BLF File Won't Load¶

    Symptom: Select BLF fails or shows error

    Solutions:

    1. Check file isn't corrupted
    2. Verify file size (<2GB recommended)
    3. Ensure sufficient memory available
    4. Try different BLF file
    5. Check file permissions

    No Signals Appearing¶

    Symptom: Signal Viewer empty during playback

    Solutions:

    1. Verify DBC file is loaded
    2. Check DBC matches log file
    3. Ensure CAN IDs align
    4. Reload the system
    5. Verify network configuration

    Playback Stutters or Freezes¶

    Symptom: Playback isn't smooth

    Solutions:

    1. Reduce number of selected signals
    2. Close unnecessary panels
    3. Clear Network Traffic periodically
    4. Reduce chart time window
    5. Close other applications

    Signal Values Don't Update¶

    Symptom: Values stay at zero or don't change

    Solutions:

    1. Verify playback is running
    2. Check signal selection
    3. Ensure DBC scaling is correct
    4. Verify signal exists in log
    5. Check frame filtering

    Chart Not Displaying¶

    Symptom: Signal Chart remains empty

    Solutions:

    1. Select signals first in Signal Viewer
    2. Ensure playback has started
    3. Check time window settings
    4. Enable Auto-scale
    5. Refresh the panel

    Export Fails¶

    Symptom: Export button doesn't work or fails

    Solutions:

    1. Stop playback first
    2. Check disk space
    3. Verify write permissions
    4. Try different format
    5. Reduce data size

    MATLAB Integration¶

    1. Export for MATLAB:
    2. Use CSV format
    3. Include timestamps
    4. Maintain signal names

    5. Preserve scaling

    6. MATLAB analysis: ```matlab % Load exported data data = readtable('cooling_signals_decoded.csv');

    % Plot temperature plot(data.Timestamp, data.Engine_Coolant_Temp); xlabel('Time (s)'); ylabel('Temperature (°C)'); title('Coolant Temperature Analysis');

    % Find peaks [peaks, locations] = findpeaks(data.Engine_Coolant_Temp); ```

    Python Analysis¶

    1. Python script example: ```python import pandas as pd import matplotlib.pyplot as plt

    # Load data df = pd.read_csv('cooling_signals_decoded.csv')

    # Analyze temperature spikes threshold = 100 # °C spikes = df[df['Engine_Coolant_Temp'] > threshold]

    # Plot results plt.plot(df['Timestamp'], df['Engine_Coolant_Temp']) plt.axhline(y=threshold, color='r', linestyle='--') plt.xlabel('Time (s)') plt.ylabel('Temperature (°C)') plt.show() ```
