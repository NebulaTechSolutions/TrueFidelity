    UI Tour¶

    This tour introduces every major area of the TrueFidelity interface so you can navigate confidently before diving into advanced workflows. All references match the default layout that appears on first launch.

    1. Header Bar¶

    The header spans the top of the window and includes:

    • Application title – Displays the name of the current project and indicates unsaved changes with an asterisk.
    • Navigation tabs – High-level categories (Home, Getting Started, Using The App, etc.) when documentation is loaded within the integrated viewer.
    • Global search – Quickly search commands, panels, signals, or documentation pages.
    • Theme toggle / update alerts – Depending on your build, you may see icons for switching themes or viewing update notifications.

    Tip

    Use Ctrl/Cmd + P to open the command palette, which searches the same command index as global search.

    2. Sidebar (Left)¶

    Project Explorer¶

    • Shows the active project, including log files, auxiliary documents, and definitions.
    • Right-click a file to reveal, remove, or open in an external editor.
    • Drag files onto the project explorer to import them quickly.

    Tools Strip¶

    • Shortcuts to panels such as Signals, Data Injection, Annotations, and Filters.
    • Hover to see tooltips; click to open the associated panel.

    Definitions¶

    • Lists imported DBC and metadata files.
    • Allows prioritizing definitions when multiple databases define the same signal.

    3. Workspace (Center)¶

    The workspace hosts dockable panels organized by workflow mode:

    System Design Mode¶

    When working with ECU systems, you'll typically see: - System Canvas – Visual editor for placing ECUs and configuring network connections - Toolbox – Palette of ECU types, networks, and components to add to your system - Properties Panel – Configuration details for selected ECU or network element - Project Explorer – File tree showing system files, configurations, and assets

    Monitoring Mode¶

    When monitoring running ECU containers: - ECU Status Panel – Overview of all running ECUs with state indicators - Console – Aggregated logs from all ECUs with filtering and search - Resource Monitor – Real-time CPU and memory graphs per ECU - Network Traffic – CAN message monitoring and analysis - Terminal – Interactive terminal access to ECU operating systems

    CAN Log Analysis Mode¶

    When analyzing CAN log files: - Timeline at the top, showing the playhead, selected range, and event markers - Messages panel on the left, presenting raw frames in timestamp order - Charts panel on the right, plotting pinned signals - Signal Viewer – Browse and search decoded signals from DBC files - Signal Player – Playback controls and injection interface

    You can rearrange panels by dragging their tabs. Docking cues appear to help position them.

    4. Right Sidebar (Inspector)¶

    Displays context-aware details about the selected item, such as:

    • Frame metadata (ID, payload, channel)
    • Signal units and scaling
    • Annotation description and tags
    • Export or injection settings

    When nothing is selected, the inspector shows quick links to documentation and keyboard shortcuts.

    5. Footer (Status Bar)¶

    Key elements include:

    • Transport controls – Play/Pause, Step Forward/Backward, Jump to Start/End, and Loop toggle.
    • Playback speed selector – Adjust between 0.25× and 8×.
    • Counters – Frames per second, queued exports, injected messages, and time remaining in the selected range.
    • Hardware indicators – Status lights for connected CAN hardware or simulation backends.

    6. Panels Overview¶

    System Design Panels¶

    • Edit ECU configuration (name, binary path, network interfaces)
    • Configure network parameters (baud rate, channel assignments)
    • Set build targets and compiler options
    • View and modify container runtime settings

    Project Explorer¶

    • Navigate system files and configurations
    • Browse firmware builds and binaries
    • Manage DBC files and definition libraries
    • Access log files and export artifacts

    ECU Monitoring Panels¶

    ECU Status Panel¶

    • Grid view of all ECUs with state indicators (running, stopped, error)
    • Quick actions: start, stop, restart, view logs
    • Resource usage summary (CPU%, memory MB)
    • Network connectivity status
    • Double-click to open ECU detail view

    Console Panel¶

    • Unified log viewer for all ECU outputs
    • Filter by ECU, log level (info, warn, error, debug)
    • Search functionality across log history
    • Color-coded log levels
    • Auto-scroll toggle and pause controls

    Resource Monitor¶

    • Real-time graphs showing CPU and memory over time
    • Per-ECU metrics with color coding
    • Historical data retention for trend analysis
    • Export metrics as CSV or PNG
    • Alert thresholds for resource exhaustion

    Network Traffic Panel¶

    • CAN message monitoring with live updates
    • Filter by message ID, channel, or ECU source
    • Message rate statistics and counters
    • Export captured traffic to log files
    • Integration with DBC definitions for signal decoding

    Terminal Panel¶

    • Multi-tab terminal interface (one per ECU)
    • Full shell access to ECU operating systems
    • Connection status indicators
    • Search functionality (Ctrl+F)
    • Terminal actions: clear, copy, reconnect

    Process Monitor¶

    • View running processes within ECUs
    • CPU and memory usage per process
    • Process tree visualization
    • Real-time updates

    CAN Log Analysis Panels¶

    Timeline¶

    • Drag to select ranges; double-click to zoom to full duration
    • Right-click to add markers or clear selections
    • Scroll wheel zooms, Shift + scroll pans horizontally

    Messages Panel¶

    • Filter frames by ID, payload, channel, or time
    • Toggle columns (timestamp, DLC, flags) via the column menu
    • Right-click to export the current filter view

    Charts Panel¶

    • Supports multiple Y-axes and color-coded signals
    • Add cursors to measure time deltas
    • Save chart screenshots via the context menu

    Signal Viewer¶

    • Browse decoded signals from DBC definitions
    • Search by signal name, message ID, unit, or tags
    • Pin frequently used signals for quick access
    • View signal metadata and scaling factors

    Signal Player¶

    • Playback controls for CAN log files
    • Timeline navigation and speed control
    • Injection interface for test message injection
    • Channel selection and filtering

    Annotations Panel¶

    • Lists bookmarks with timestamps, tags, and descriptions
    • Double-click to jump the timeline to the annotation
    • Export annotations as CSV or include them in reports

    Data Injection Panel¶

    • Configure message templates, schedules, and profiles
    • Start/stop injections; monitor frame counts and errors
    • Import or export injection profiles for reuse

    7. Keyboard Navigation Highlights¶

    Panel Toggles¶

    • Ctrl/Cmd + 1 – Toggle Signals/ECU Status panel (context-dependent)
    • Ctrl/Cmd + 2 – Toggle Messages/Console panel
    • Ctrl/Cmd + 3 – Toggle Charts/Resource Monitor panel
    • Ctrl/Cmd + 4 – Toggle Data Injection/Terminal panel
    • Ctrl/Cmd + W – Close the active panel tab
    • Alt + Arrow keys – Switch between docked panel tabs

    System Design¶

    • Ctrl/Cmd + N – Create new system
    • Ctrl/Cmd + O – Open system
    • Ctrl/Cmd + S – Save system
    • Delete – Remove selected ECU or network element (when canvas focused)

    CAN Playback¶

    • Space – Play/Pause playback
    • Arrow Left/Right – Step backward/forward
    • Home/End – Jump to start/end of log
    • Shift + C – Add cursor to chart

    8. Customizing the UI¶

    • Drag panel edges to resize.
    • Use View → Reset Layout to recover from accidental rearrangements.
    • Save layout variants inside a project for quick context switching (e.g., Analysis vs. Injection layout).
    • Access Settings → Theme to choose dark/light or Catppuccin-inspired color schemes if available.

    9. Embedded Documentation Viewer¶

    If the documentation is packaged with the app, selecting Help → User Guide opens this site within an embedded view, allowing cross-referencing while you work. The viewer retains navigation history and respects the same theme colors as the main app.
