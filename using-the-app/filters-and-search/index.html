    Filters & Search¶

    Filtering and searching allow you to isolate critical frames, signals, and time windows quickly. This guide covers message filters, signal search, saved queries, and best practices for building reusable filters.

    Messages Panel Filters¶

    Quick Filters¶

    • ID filter: Enter numeric IDs in hex (0x123) or decimal (291). Use comma-separated lists for multiple IDs.
    • Payload search: Type a hex sequence (AA FF) to match payload segments. Wildcards (??) match any nibble.
    • Channel filter: Choose specific channels (e.g., can0) using the channel dropdown.
    • Time range: Use the time slider or enter start/end times to focus on a segment.

    Advanced Filter Builder¶

    1. Click the Filter button in the Messages toolbar.
    2. Build conditions using AND/OR groups.
    3. Available fields: ID, DLC, Payload, Channel, Timestamp, Flags, Signal values (when decoded).
    4. Save the filter with a descriptive name (e.g., Safety_Critical_IDs).

    Example filter:

    (ID == 0x123 OR ID == 0x456) AND Payload starts with AA AND Timestamp between 10s and 60s

    Saved filters appear in the dropdown for one-click reuse.

    Signal Search & Watch Lists¶

    • Open the Signals panel and use the search box to find signals by name, unit, or DBC comment.
    • Filter by source definition file to reduce results when multiple DBCs overlap.
    • Pin signals to add them to the watch list; pinned signals stay visible across sessions.
    • Use tags (configured in your definitions) to group signals (e.g., powertrain, braking).

    Tip

    When searching for diagnostic PIDs, prefix with 0x to match exactly; otherwise partial text search returns broader results.

    Timeline Search¶

    • Press Ctrl/Cmd + F to open the timeline search bar.
    • Enter a timestamp (00:10:25.500) or relative offset (+5s, -3s) to jump the playhead.
    • Search for annotations by title/tag (e.g., tag:spike).

    Global Search (Command Palette)¶

    • Press Ctrl/Cmd + P.
    • Type commands (export segment, add annotation), panel names (charts, signals), or documentation topics (data injection).
    • Use arrow keys to select results and press Enter to execute.

    Combining Filters with Timeline Ranges¶

    1. Select a range on the timeline.
    2. Enable Filter to Range (toolbar icon or Ctrl/Cmd + Shift + F).
    3. Apply message filters to focus on IDs or payloads within that range.
    4. Charts and statistics update instantly based on the filtered data.

    Saved Filter Sets¶

    • Use Filter Sets to bundle multiple filters (ID + payload + time) into a single preset.
    • Apply a filter set with one click from the toolbar.
    • Export/import filter sets (*.tffilter) to share with colleagues.

    Highlighting Frames¶

    • Enable Highlight rules in the Messages panel to color frames that match certain conditions (e.g., errors, thresholds).
    • Configure the rule color, criteria, and optional sound alert.

    Search in Exports¶

    • When exporting CSV, enable Include filters in metadata so recipients know which criteria were used.
    • Report Builder automatically lists active filters in the summary page.

    Diagnostics for Filters¶

    • Use the Filter Inspector (View → Filter Inspector) to see which filters are active across panels.
    • Temporarily disable a filter via the toggle next to its name to compare results quickly.

    Managing Performance¶

    • Avoid stacking too many complex OR conditions; break them into separate filters and apply sequentially.
    • Use numeric comparisons (>, <, between) instead of large ID lists when possible.
    • For enormous datasets, apply filters before enabling heavy visualizations.

    Example Workflows¶

    Locate Brake Events¶

    1. Filter IDs to known brake-related frames.
    2. Apply payload rule for the bit indicating brake pedal pressed.
    3. Filter time range to 0–300 s.
    4. Add annotation template (Brake Event) for quick tagging.

    Isolate Fault Codes¶

    1. Search signals for DTC or fault.
    2. Pin relevant signals and add them to Charts.
    3. Create a filter for payload patterns that correspond to diagnostic trouble codes.
    4. Export only frames matching the payload filter.

    Validate New Firmware¶

    1. Load baseline and new firmware logs.
    2. Apply the same filter set to both sessions.
    3. Use charts to compare key signals; highlight mismatched ranges with annotations.

    Keyboard Shortcuts¶

    Action Shortcut
    Toggle Messages filter bar Ctrl/Cmd + F
    Clear all filters Ctrl/Cmd + Shift + C
    Focus global search Ctrl/Cmd + P
    Add current filter to set Ctrl/Cmd + Shift + S

    Troubleshooting Filters¶

    Symptom Fix
    Filter returns no frames Relax conditions or check for inverted comparisons (e.g., < vs <=).
    Filters reset after restart Ensure you saved the filter; autosave must be enabled or manually save the project.
    Performance slows dramatically Combine filters or simplify payload rules; consider limiting to a smaller range.
    Filters not applied to charts Ensure Filter to Range is active or add the filter directly in the Charts panel’s data source settings.
