    Annotations & Bookmarks¶

    Annotations and bookmarks capture insights while you analyze playback. They add structure to reports, help teams collaborate, and make exporting findings quick.

    Annotations vs. Bookmarks¶

    • Bookmark: Quick marker with a title and timestamp. Ideal for rapid flagging.
    • Annotation: Expanded note with description, tags, severity, attachments, and linked segments.
    • Both appear on the timeline as markers and in the Annotations panel for navigation.

    Creating Annotations¶

    1. Pause playback at the moment of interest.
    2. Press A or click the Add Annotation button in the Annotations panel.
    3. Fill in the details:
    4. Title – Keep concise but descriptive.
    5. Description – Provide context, observed behavior, expected behavior, and next steps.
    6. Tags – Add keywords like safety, powertrain, regression.
    7. Severity – Optional field if your build includes severity levels.
    8. Attachments – Link to external documents or evidence, stored relative to the project.
    9. Click Save. The annotation now appears in the list and on the timeline.

    Tip

    Use copy/paste for annotations to duplicate structure across similar events—handy when logging repeated occurrences.

    Bookmark Workflow¶

    • Press B to drop a bookmark at the current time.
    • Rename directly in the timeline or in the Annotations panel.
    • Upgrade a bookmark to a full annotation later by adding description and tags.

    Managing Annotations¶

    Sorting & Filtering¶

    • Sort by timestamp, title, severity, or tag.
    • Filter by tag to focus on specific themes (e.g., noise, CAN-error).
    • Combine filters with timeline ranges to review annotations within a particular segment.

    Editing & Deleting¶

    • Select an annotation and click Edit to update fields.
    • Use Duplicate to create a copy at the same timestamp (change the tags to differentiate).
    • Delete with Del or the context menu (action is reversible via undo until you close the project).

    Linking to Segments¶

    • With a range selected, open an annotation and choose Link to current segment. This ties the note to that exact duration for exports.
    • Multiple annotations can link to the same segment—ideal for collaborative reviews.

    Annotation Templates¶

    1. Navigate to Annotations → Templates.
    2. Create templates for recurring issues (e.g., “Spike detected”, “CAN timeout”).
    3. Predefine tags, severity, and description prompts.
    4. When adding a new annotation, choose a template to prefill fields. Customize as needed before saving.

    Exporting Annotations¶

    • CSV Export: Click Export → CSV to generate a spreadsheet including timestamp, title, tags, and description.
    • Report Builder: Add selected annotations to the report queue. The builder inserts them into summary tables and narrative sections.
    • Screenshot integration: Use the screenshot button within the annotation to capture the current chart or panel view; the image links to the annotation.

    Collaboration Tips¶

    • Assign tags to identify the author (tag:alice).
    • Use severity levels (critical, major, minor) to prioritize follow-up work.
    • Include links to issue trackers in the description for traceability.
    • Hold review sessions by filtering annotations to a specific tag (e.g., ready-for-review).

    Keyboard Shortcuts¶

    Action Shortcut
    Add annotation A
    Add bookmark B
    Toggle Annotations panel Ctrl/Cmd + 6 (configurable)
    Jump to next/previous annotation Alt + ↓ / Alt + ↑
    Filter annotations by current tag Ctrl/Cmd + Shift + T

    Troubleshooting¶

    Issue Fix
    Annotations missing after reopen Ensure autosave completed or manually save the project before closing.
    Tags inconsistent Establish a tag naming convention and share it in team documentation.
    Export missing screenshots Ensure screenshots were saved to a path accessible relative to the project before exporting.
    Timeline cluttered Hide annotation markers temporarily via the timeline options menu.

    Integrating With Reports¶

    • Use Report Builder to include annotation tables automatically.
    • Add context paragraphs to the description field; they appear verbatim in the report.
    • Attach charts or links to rerun scripts to support each annotation’s findings.
