    Concepts & Terminology¶

    This glossary clarifies the key terms TrueFidelity users encounter when navigating projects, decoding signals, and injecting data. Understanding these concepts up front reduces onboarding friction and ensures consistent communication across teams.

    Core Objects¶

    System Design¶

    • System – A configuration defining multiple ECUs, their connections, network topologies, and build settings. Systems are saved as .tfsystem files.
    • ECU (Electronic Control Unit) – A virtual ECU instance running in a Docker container. Each ECU has a name, binary, network interfaces, and runtime configuration.
    • System Canvas – The visual workspace for designing ECU topologies by dragging and connecting components.
    • Network Edge – A connection between ECUs representing a CAN bus, Ethernet link, or other network interface.
    • Toolbox – A panel containing reusable ECU types, network components, and gateway elements that can be dragged onto the canvas.

    Projects & Files¶

    • Project – A container that remembers open files, panel layouts, watch lists, annotations, and definitions. Projects are saved as .tfproj files referencing data stored elsewhere.
    • Workspace/Layout – The arrangement of panels in the main window (timeline, charts, messages, monitoring panels). Layouts can be saved within a project and reused.
    • Panel – Any dockable UI component such as System Canvas, ECU Status, Console, Messages, Charts, Signals, Annotations, or Data Injection. Panels can be docked, floated, or collapsed.

    CAN Log Analysis¶

    • Timeline – The visual bar representing elapsed time in the loaded CAN log(s). The playhead moves along it, and users can select ranges for export or looping.
    • Channel/Bus – A logical CAN interface (e.g., can0, can1). When injecting or filtering data, you select which channel to use.
    • Segment/Range – A highlighted portion of the timeline defined by dragging or specifying start/end times. Segments drive exports, looped playback, filtering, and reporting.

    Data Terminology¶

    • CAN Frame – A single message on the CAN bus, defined by ID, DLC, payload, timestamp, and optional flags (extended ID, RTR).
    • Signal – A decoded value derived from a CAN payload using a definition file (usually DBC). Signals have units, scaling, and offsets.
    • Message Definition – Metadata that explains how to decode a CAN frame into one or more signals. Stored in DBC or other definition formats.
    • Event Marker – A bookmark or annotation added at a specific time to capture context, issues, or observations.
    • Counters/Statistics – Real-time metrics displayed during playback (messages per second, dropped frames, current time).

    File Types¶

    • Log File – Recorded CAN traffic. Common formats: MDF4 (.mf4, .mdf), BLF (.blf), ASC (.asc), CSV.
    • Definition File – DBC (.dbc), ARXML, or other mapping files that define signals and scaling for raw frames.
    • Export File – Data generated from TrueFidelity (CSV segments, charts as images, annotation reports, injection profiles).
    • Profile – Serialized configuration for injection schedules or saved filters (depending on feature).

    Playback Concepts¶

    • Playback Speed – The rate at which the timeline advances relative to recorded time (0.25× to 8×).
    • Loop Mode – Repeats the selected range on the timeline until disabled. Useful for investigating short events.
    • Cursors – Vertical markers in Charts used to measure time difference between two points.
    • Auto-Scale – Chart mode that adjusts Y-axis bounds automatically based on visible data. Disable to compare runs with consistent scaling.
    • Filter-to-Range – Restricts visible data (Messages, Charts) to the currently selected timeline range.

    Injection Concepts¶

    • Injection Panel – Panel used to schedule CAN frames for transmission alongside playback or live monitoring.
    • Message Template – Defines ID, DLC, payload, and optional flags for a single injected message.
    • Schedule – Timing definition for an injection (once, interval, burst, ramp).
    • Profile – Grouping of templates and schedules saved for reuse. Profiles can be exported/imported for collaboration.
    • Safety Guard – A mechanism that warns or blocks injection when certain risk conditions are met (e.g., missing hardware acknowledgement).

    Project Collaboration Terms¶

    • Pinned Signal – A signal flagged in the Signals panel to keep it readily available across sessions.
    • Watch List – A curated list of pinned signals displayed in the sidebar for quick charting.
    • Annotation Tag – A keyword applied to annotations (e.g., safety, powertrain, regression). Tags help filter notes and build thematic reports.
    • Shared Layout – A project layout exported and imported by teammates to synchronize workspace arrangement.
    • Session Log – Auto-generated background logs capturing errors, warnings, and telemetry (if enabled). Access via Help → Show Logs.

    Monitoring & Runtime Terminology¶

    • ECU Status – Current state of an ECU container (Running, Stopped, Starting, Error). Displayed in ECU Status panel with color-coded indicators.
    • Container – Docker container instance running an ECU binary. Each ECU in a system runs as a separate container.
    • Console Logs – Aggregated standard output and error streams from all ECU containers. Viewable in the Console panel with filtering.
    • Resource Metrics – CPU and memory usage data collected from running ECU containers. Displayed as real-time graphs in Resource Monitor panel.
    • Terminal Access – Interactive shell access to ECU operating systems through the Terminal panel. Provides command-line debugging and configuration.
    • Network Traffic – CAN messages flowing between ECUs. Captured and displayed in real-time in the Network Traffic panel.
    • Process Monitor – View of running processes within ECU containers, including CPU/memory per process.

    Status Indicators¶

    • Activation Badge – An icon in the status bar showing license status (green for activated, amber for trial, red for expired).
    • ECU Status Indicators – Color-coded states in ECU Status panel: green (running), yellow (starting), red (error), gray (stopped).
    • Terminal Connection Status – Indicators in Terminal panel: green (connected), yellow (connecting), red (connection failed).
    • Autosave Indicator – Flashes when the project autosaves; disable or adjust frequency in Settings → General.
    • Hardware Status – For systems with CAN hardware integration, device state is shown in the footer (connected, disconnected, error).

    Keyboard Shortcut Vocabulary¶

    • Toggle Panels – Use Ctrl/Cmd + number (e.g., 1 for Signals, 2 for Messages) to show or hide panels quickly.
    • Command Palette – Press Ctrl/Cmd + P to bring up a search-driven command list (if enabled in your build).
    • Cursors – Add with Shift + C (configurable) to mark points on charts.

    Reporting & Exports¶

    • Report Builder – A guided wizard that combines annotations, screenshots, and summary metrics into PDF/HTML outputs.
    • Batch Export Queue – A task list that processes multiple timeline segments or files overnight.
    • Snapshot – A quick capture of the current chart or panel state saved as an image.

    Troubleshooting Vocabulary¶

    • Safe Mode – Launch flag that disables hardware acceleration (--safe-mode). Use for diagnosing display issues.
    • Verbose Logging – Start TrueFidelity with --verbose to capture additional diagnostics in log files.
    • Compatibility Mode – A playback option that relaxes parsing strictness for older log formats; configure under Settings → Data.

    Using This Glossary¶

    • Keep this page open in a split view while onboarding new team members.
    • Link definitions in documentation or runbooks using anchors (e.g., #signal).
    • Encourage consistent terminology when filing support tickets to speed up triage.
