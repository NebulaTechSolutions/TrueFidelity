    First Launch & Activation¶

    Use this guide immediately after installing TrueFidelity to complete activation, set defaults, and verify your environment. All steps are user-facing and omit internal implementation details.

    Launching the App for the First Time¶

    1. Start TrueFidelity from your platform’s application menu.
    2. A splash screen appears for a few seconds; then the main window opens with the default layout.
    3. If the splash screen hangs for longer than 30 seconds, see the troubleshooting section below.

    License Agreement¶

    • Review the license agreement presented on first launch.
    • Click I agree to continue. If you do not agree, close the application—no changes are written.
    • For an enterprise distribution, ensure legal teams capture the agreement text outside the application for auditing.

    Activation Options¶

    TrueFidelity can operate with online or offline activation depending on your deployment.

    Online Activation¶

    1. Locate your license key (usually delivered via email or portal).
    2. Paste the key into the activation prompt.

    1. If your network requires a proxy, click Proxy Settings, provide the host/port, and credentials if needed.

    1. Click Activate. Online activation completes within seconds.
    2. The license status indicator in the lower-right corner turns green when successful.

    Offline Activation¶

    1. On a connected machine, generate an offline license file from your distribution portal.
    2. Transfer the file to the air-gapped workstation.
    3. Choose Offline activation and select the license file.
    4. Confirm the activation summary shows your organization name and expiration (if any).

    License grace period

    If you accidentally close the activation window, TrueFidelity allows a short trial period before requiring activation. A banner will remind you until activation is complete.

    Initial Setup Wizard¶

    After activation you will see a short wizard to set preferred defaults. These choices can be changed later under Settings.

    1. Default project folder: Select a location on fast storage (SSD). All new projects use this path unless you override it.

    1. Default log folder: Choose where incoming CAN logs are usually stored.

    1. Default export folder: Define a destination for exported segments, reports, and annotations.

    1. Telemetry preference: Opt in or out of anonymous usage metrics (if offered in your distribution).

    1. Automatic updates: Keep enabled for most environments; disable if you operate under release control.

    Verifying Hardware and Permissions¶

    • Open Help → About to confirm version number and license details.
    • Create a test project (File → New Project), save it in your project folder, then close and reopen the project to ensure read/write access.
    • Open Settings → General and verify the UI language, theme, and time display preferences.
    • Navigate to Settings → Playback to confirm default playback speed, looping behavior, and cursor display options.
    • If you plan to inject data, ensure the user has necessary OS-level permissions for network interfaces.

    Default Layout Walkthrough¶

    The default workspace adapts based on your workflow mode:

    System Design Mode (when creating new systems)¶

    • System Canvas – Visual editor for placing ECUs and configuring networks
    • Toolbox – Component library for adding ECUs and network elements
    • Properties Panel – Configuration for selected ECU or network
    • Project Explorer – File tree for system files and assets

    Monitoring Mode (when systems are running)¶

    • ECU Status – Overview of running ECU containers
    • Console – Aggregated logs from all ECUs
    • Resource Monitor – Real-time CPU and memory graphs
    • Network Traffic – CAN message monitoring

    CAN Log Analysis Mode (when analyzing log files)¶

    • Timeline – Playback timeline with playhead and selection
    • Messages – Raw CAN frames in timestamp order
    • Charts – Signal visualization and plotting
    • Signals – Browse decoded signals from DBC files

    Take a moment to expand and collapse panels, open the context menus, and familiarize yourself with the UI. The UI Tour page provides screenshots and descriptions of each region.

    First Run Test Workflows¶

    Option A: CAN Log Analysis Test¶

    1. Open a sample log: Use File → Open to select a known-good CAN log file.
    2. Play: Press the space bar or click the play button. Confirm the timeline moves and message counters update.
    3. Seek: Click at different points on the timeline and watch the data in Messages and Charts change.
    4. Add a signal: Open the Signals panel, search for a familiar signal, and pin it to the watch list.
    5. Plot: Add the signal to Charts to confirm decoding is working.
    6. Export test segment: Drag-select five seconds on the timeline, then export to CSV and verify the file is written to your export folder.
    7. Close project: Ensure there are no errors when closing the application; reopen it to confirm your preferences persist.

    Option B: System Design Test¶

    1. Create a new system: Use File → New to create a test system.
    2. Add an ECU: Drag an ECU from the Toolbox onto the System Canvas.
    3. Configure properties: Select the ECU and verify the Properties panel shows editable fields.
    4. Save system: Press Ctrl/Cmd + S and confirm the system file is created.
    5. Start ECU (if Docker available): Right-click the ECU and select Start to verify container launch.
    6. View monitoring: Open ECU Status and Console panels to see runtime data.
    7. Close system: Ensure no errors when closing; reopen to verify persistence.

    Note

    The System Design workflow requires Docker Desktop to be running for ECU containers to start. CAN Log Analysis works independently of Docker.

    Troubleshooting First Launch¶

    Issue Suggested Action
    App window stays blank Launch from a terminal with truefidelity --disable-gpu (Linux/macOS) or "TrueFidelity.exe" --disable-gpu (Windows). Update GPU drivers afterwards.
    Activation fails Check internet connectivity or proxy credentials. If activating offline, verify the license file matches the machine.
    Wizard cannot save defaults Confirm write access to the selected folders or choose alternate directories.
    Missing panels Select View → Reset Layout to restore the default arrangement.
    Fonts or UI scaling is off Adjust OS display scaling or use Settings → Accessibility → UI scaling inside the app.

    Collecting Support Information¶

    If you encounter an issue during first launch:

    1. Use Help → Show Logs to open the log directory.
    2. Compress the most recent log files.
    3. Capture screenshots of any error dialogs.
    4. Document the steps taken and include the app version from Help → About.
    5. Submit the bundle to your support channel.
