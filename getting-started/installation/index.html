    Installation¶

    This guide walks through installing the TrueFidelity desktop application on Windows, macOS, and Linux. It also covers system requirements, preparatory steps, updates, uninstallation, and troubleshooting. Share this page with end users; it contains no internal implementation details.

    System Requirements¶

    • Operating systems: Windows 10/11 (x64), macOS 12+ (Intel or Apple Silicon), Linux (x86_64) with modern desktop environment
    • CPU: Quad-core 2.5 GHz or better (8+ cores recommended for large logs)
    • Memory: Minimum 8 GB; 16 GB or more for long sessions and simultaneous exports
    • Storage: 1 GB free for the app plus capacity for log archives and exports (SSD strongly recommended)
    • Graphics: GPU with updated drivers for hardware acceleration (integrated GPUs are supported)
    • Permissions: Ability to install desktop applications and read/write to your log and export folders

    Downloading the Installer¶

    1. Visit your official distribution portal or shared release storage.
    2. Choose the installer for your platform:
    3. Windows: .exe
    4. macOS: .dmg
    5. Linux: .AppImage, .deb, or .rpm
    6. Verify the filename and checksum (if provided). For the highest assurance, compare the SHA-256 checksum published with the release.
    7. Place the installer on a local disk before running (avoid network shares for initial installation).

    Keep past installers

    If you operate in a validated environment, archive the installers you have approved. This lets you revert quickly if a new release requires certification.

    Windows Installation¶

    1. Right-click the installer and select Run as administrator (optional but helps when installing for all users).
    2. If Windows SmartScreen prompts you, choose More info → Run anyway (verify the publisher matches your organization).

    1. Follow the setup wizard:
    2. Scope: install for Just me or All users.
    3. Destination folder: the default C:\Program Files\TrueFidelity works for most deployments.
    4. Shortcuts: enable Start menu and desktop entries for quick access.

    1. Click Install and wait for completion.

    1. Launch TrueFidelity from the Start menu to verify a successful install.

    Windows Troubleshooting¶

    • SmartScreen block persists: Ensure you downloaded from a trusted source or contact IT to add the certificate to your allow list.
    • App does not launch: Check antivirus quarantine logs and add the installation folder to your safe-list if necessary.
    • Silent install: Use TrueFidelity-x.y.z.exe /S /allusers to script deployment.

    macOS Installation¶

    1. Double-click the downloaded .dmg file to mount it.

    1. Drag the TrueFidelity application into the Applications folder shortcut.
    2. Eject the disk image.

    1. Launch the app from Launchpad or Spotlight.

    1. If macOS warns the app is from an unidentified developer, right-click the app, choose Open, then click Open again. Alternatively, go to System Settings → Privacy & Security and allow the app.

    macOS Troubleshooting¶

    • App immediately closes: Remove any quarantine attributes using xattr -rd com.apple.quarantine /Applications/TrueFidelity.app.
    • Rosetta prompt: Apple Silicon systems may request Rosetta if Intel binaries are used; allow installation when prompted.

    Linux Installation¶

    AppImage¶

    1. Mark the file executable: chmod +x TrueFidelity-x.y.z.AppImage.
    2. Run the AppImage: ./TrueFidelity-x.y.z.AppImage.
    3. Accept integration prompts if you want menu entries.

    Debian/Ubuntu (.deb)¶

    sudo apt install ./truefidelity_x.y.z_amd64.deb

    RHEL/Fedora (.rpm)¶

    sudo dnf install ./truefidelity-x.y.z.x86_64.rpm

    Linux Troubleshooting¶

    • Missing libraries: Use your package manager to install libfuse, gtk3, or other prerequisites indicated in the terminal.
    • Wayland issues: If the UI does not render, try launching with --disable-gpu or from an X11 session.

    First Launch Checklist¶

    • Confirm the splash screen appears and transitions to the main window within 30 seconds.
    • Approve any firewall prompts to allow local networking features (used for internal services).
    • Verify you can open the Help → About dialog to check version and license status.
    • Create a test project in a temporary folder and delete it afterwards to ensure write permissions.

    Updating TrueFidelity¶

    • Automatic check: If enabled, the app will notify you about updates. Follow prompts to download the newest installer.
    • Manual update: Download the latest installer and run it over the existing installation. Your settings, activation, and projects remain intact.
    • Version pinning: When you must stay on a particular version, disable auto-updates in Settings → Updates and retain the approved installer.

    Reboot after major upgrades

    If the upgrade includes driver-level components, reboot to ensure hardware integrations reload correctly.

    Uninstalling¶

    • Windows: Settings → Apps → Installed apps → TrueFidelity → Uninstall. Optional: remove leftover data in %APPDATA%\TrueFidelity.
    • macOS: Drag the app from Applications to the Trash. Delete support files in ~/Library/Application Support/TrueFidelity if you want a clean slate.
    • Linux (package): sudo apt remove truefidelity or sudo dnf remove truefidelity.
    • Linux (AppImage): Delete the file plus any desktop integration entries created under ~/.local/share.

    Troubleshooting Common Issues¶

    Symptom Recommended Action
    Installer fails mid-way Temporarily disable antivirus or install using admin privileges.
    App window stays blank Launch with --disable-gpu, update graphics drivers, or check for incompatible overlays.
    Can’t write to default folders Choose user-writable paths in the setup wizard or adjust folder permissions.
    Proxy/Firewall blocks activation Configure proxy settings in the activation dialog or obtain an offline activation file.

    Collecting Diagnostic Logs¶

    1. Launch the app with verbose logging (see Help → Show Logs for file locations).
    2. Reproduce the issue.
    3. Compress the logs directory and attach it when requesting support.
