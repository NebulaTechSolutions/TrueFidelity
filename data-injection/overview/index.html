    Data Injection Overview¶

    Data injection lets you transmit CAN frames from TrueFidelity while you monitor playback or live systems. Use it to simulate sensors, reproduce edge cases, or validate control strategies in controlled environments.

    When to Use Injection¶

    • Simulating missing sensors or altered signals during bench testing
    • Reproducing field issues without waiting for the exact conditions
    • Stress-testing ECUs by increasing message rates
    • Validating firmware updates that rely on specific message sequences

    Safety First¶

    • Perform injection only in controlled test environments.
    • Keep a quick way to disconnect hardware or stop injection.
    • Document all injected messages to ensure traceability and avoid confusion during analysis.

    Hardware & Permissions¶

    • Ensure your machine has permissions to access the CAN interface (SocketCAN on Linux, driver packages on Windows).
    • Confirm your hardware supports injection and is not set to monitor-only mode.
    • In multi-user environments, coordinate access to avoid conflicting injections.

    Panel Overview¶

    Open the Data Injection panel from the sidebar. The panel includes:

    1. Channel selector – Choose the CAN bus (can0, can1, etc.).
    2. Message list – Each entry defines ID, DLC, payload, and flags.
    3. Schedule controls – Configure timing (once, interval, burst, ramp).
    4. Counters – Sent frames, failures, current rate, elapsed time.
    5. Status log – Inline log showing start/stop events and errors.

    Creating a Message Template¶

    1. Click Add Message.
    2. Enter ID (hex or decimal) and select standard vs. extended.
    3. Set DLC (0–8 for classic CAN).
    4. Enter payload as hex bytes (AA FF 00 01 ...).
    5. Optional: Set flags (RTR, BRS) depending on your build.

    Scheduling Injection¶

    • Once: Sends the message once after a specified delay.
    • Interval: Repeats at a fixed rate (e.g., every 100 ms).
    • Burst: Sends N frames rapidly, then pauses for M seconds.
    • Ramp: Gradually increases or decreases the rate across a defined duration.
    • Custom script: Some deployments allow importing a script for complex patterns.

    Configure the schedule in the lower portion of the panel; preview charts show expected timing.

    Starting & Monitoring Injection¶

    1. Review hardware connection and message templates.
    2. Click Start. The status indicator turns green.
    3. Monitor counters to confirm messages are being sent.
    4. Watch the Messages panel to verify the injected frames appear as expected.
    5. Click Stop to end injection immediately.

    Warning

    If you see transmission errors or failures incrementing rapidly, stop injection and inspect hardware or bus load.

    Profiles¶

    • Use Save Profile to store a set of messages and schedules as a .tfinject file.
    • Load profiles to reproduce scenarios quickly (e.g., ABS_failover, Sensor_spike_test).
    • Profiles capture message templates, schedules, and channel selection.

    Integration with Playback¶

    • Run injection while playing back logs to blend simulated messages with recorded data.
    • Use different channels to separate injected frames from recorded ones if needed.
    • Annotate injection start/stop times to maintain traceability.

    Logs & Audit Trail¶

    • The injection panel logs start/stop events, errors, and parameter changes.
    • Export injection logs from the panel or Help → Show Logs for audits.
    • Include injection profiles and logs in project archives.

    Troubleshooting Injection¶

    Issue Solution
    Messages not appearing Verify the correct channel is selected and not in listen-only mode. Check for bus-off conditions.
    Injection stops unexpectedly Look for hardware errors in the status log; ensure other tools are not controlling the interface.
    Rate inaccurate Adjust schedule settings; confirm the system clock is stable and the machine is not under heavy CPU load.
    Unable to start Confirm activation license includes injection features and the profile is valid.
