    Release Channels & Updates¶

    Stay current with TrueFidelity by understanding release channels, update cadence, and how to plan upgrades.

    Release Channels¶

    Channel Description Recommended For
    Stable Fully validated releases with complete QA. Production teams, regulated environments.
    Beta Includes upcoming features with limited validation. Power users evaluating new functionality.
    Nightly / Insider Latest builds with experimental changes. Tooling teams, early adopters testing fixes.

    Confirm which channels are available to your organization—access may depend on licensing.

    Version Numbering¶

    TrueFidelity uses semantic versioning: MAJOR.MINOR.PATCH. - MAJOR – Significant changes; may include breaking updates or new hardware support. - MINOR – Feature additions and improvements. - PATCH – Bug fixes and small enhancements.

    Update Process¶

    1. Review release notes for highlights, breaking changes, and upgrade instructions.
    2. Back up critical projects and settings (Settings → Backup).
    3. Download the installer from your authorized portal.
    4. Run the installer; existing settings and projects remain intact.
    5. Restart TrueFidelity and verify the version under Help → About.

    Tip

    Schedule upgrades during low-activity periods. For shared labs, coordinate with the team and document the change in your tool calendar.

    Rollback Strategy¶

    • Keep previous installers archived (e.g., in your software repository).
    • If issues arise post-upgrade, uninstall the current version and reinstall the prior stable build.
    • Restore backed-up settings if necessary.

    Testing & Validation¶

    • For regulated environments, run smoke tests on staging machines before rolling out widely.
    • Use representative projects and logs to ensure compatibility.
    • Document test results and approval before updating production systems.

    Automatic Update Checks¶

    • Enable or disable under Settings → Updates.
    • Automatic checks notify you of new builds but do not install automatically unless configured.
    • Beta/nightly channels may require manual opt-in.

    Communication¶

    • Maintain an internal changelog summarizing updates, test results, and deployment plan.
    • Notify stakeholders (test teams, developers, support staff) about planned upgrades.
    • Provide rollback instructions in case of unexpected behavior.

    Frequently Asked Questions¶

    Q: Can we skip versions?
    A: Yes, but review release notes for cumulative changes. Some features depend on intermediate updates.

    Q: How do we access beta/nightly builds?
    A: Contact your license administrator or vendor representative to enable additional channels.

    Q: Are settings compatible across versions?
    A: Minor and patch releases remain compatible. Review release notes for any major-version migration steps.
