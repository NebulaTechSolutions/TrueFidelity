    Settings Reference¶

    Customize TrueFidelity to match your workflow using the Settings dialog. This reference covers each category and notable options.

    General¶

    • Language – UI language; restart may be required.
    • Theme – Choose between light, dark, or custom palettes (e.g., Catppuccin).
    • Autosave interval – Frequency at which projects auto-save (1–30 minutes). Set to 0 to disable.
    • Recent items limit – Number of recent projects and files shown.
    • Telemetry – Opt in/out of anonymous usage metrics (if offered in your distribution).

    Workspace¶

    • Default layout – Layout applied to new projects. Choose from saved layouts.
    • Docking behavior – Adjust drag sensitivity to prevent accidental undocking.
    • Panel animations – Enable/disable transition animations when docking panels.

    Playback¶

    • Default speed – Initial playback rate (1× recommended).
    • Resume playback on open – If enabled, playback starts when you reopen a project.
    • Loop selection by default – Automatically loop selected ranges.
    • Buffer size – Number of frames cached during playback; increase for smoother playback on high-IO systems.

    Data & Import¶

    • Copy imported files – Copy logs into the project folder or reference original path.
    • Normalize timestamps – Automatically convert relative timestamps to absolute when importing.
    • Default channel naming – Set naming scheme for new channels (e.g., CAN_A, CAN_B).
    • CSV column presets – Save column mappings for recurring CSV imports.

    Charts¶

    • Theme – Choose color palettes for signals (e.g., color-blind friendly)."
    • Anti-aliasing – Smooth lines for readability; disable for performance.
    • Max samples – Limit number of points drawn to maintain performance.
    • Cursor appearance – Customize color and width of measurement cursors.

    Messages¶

    • Timestamp display – Choose absolute, relative, or delta timestamps.
    • Payload formatting – Hex with spaces, continuous hex, or ASCII preview.
    • Auto-freeze threshold – Optionally freeze scrolling when message rate exceeds a set value.
    • Persist filters – Reapply last-used filters when reopening a project.

    Annotations¶

    • Default tags – Pre-populate tags for new annotations (e.g., needs-review).
    • Severity levels – Configure labels and colors (critical, major, minor).
    • Template directory – Path where annotation templates are stored.

    Data Injection¶

    • Default channel – Preselect a channel when opening the injection panel.
    • Safety prompts – Enable extra confirmations before starting injection.
    • Rate limiter – Cap maximum injection frequency for safety.
    • Profile storage – Default path for .tfinject files.

    Exports¶

    • Default export folder – Directory used when no path is specified.
    • Filename template – Use tokens like {project}, {range}, {timestamp} to automate naming.
    • CSV delimiter – Choose comma, semicolon, or tab.
    • Include filter summary – Append active filters to export metadata.

    Reports¶

    • Default template – Template preselected in Report Builder.
    • Branding assets – Logos or cover images applied to exported reports.
    • Auto-embed screenshots – Automatically include latest chart captures when building reports.

    Shortcuts¶

    • Customize keyboard mappings for commands, panels, and playback.
    • Export/import shortcut profiles (.tfkeys).

    Accessibility¶

    • High-contrast focus – Enhances focus outlines for keyboard navigation.
    • UI scaling – Scale the interface (80–150%).
    • Reduced motion – Disable panel animations for sensitive users.
    • Screen reader hints – Enable additional UI hints for screen reader compatibility.

    Updates¶

    • Auto-check for updates – Enable to receive release notifications.
    • Channel selection – Choose stable, beta, or nightly channels (requires permission).
    • Download path – Location where installers are stored.

    Diagnostics¶

    • Verbose logging – Increase log detail for troubleshooting (impacts performance slightly).
    • Log retention – Number of days to keep log files before automatic cleanup.
    • Hardware monitoring – Toggle display of CPU/GPU utilization overlay.

    Reset & Recovery¶

    • Restore defaults – Reset all settings to factory values.
    • Backup settings – Export current settings to JSON for safekeeping.
    • Import settings – Apply settings from a previously saved file.

    Tips for Managing Settings¶

    • Document organizational defaults in a shared guideline.
    • Store exported settings alongside projects to recreate environments quickly.
    • Review settings quarterly to adopt new features introduced by updates.
