    Glossary¶

    Key terms used throughout the TrueFidelity documentation.

    A¶

    Annotation – A user-created note tied to a specific timestamp or range, including title, description, tags, and optional attachments.

    Autosave – Automatic saving of project state at defined intervals.

    B¶

    Bookmark – A lightweight marker on the timeline with a title and timestamp.

    Burst (Injection) – Schedule sending multiple messages rapidly followed by a pause.

    C¶

    CAN Frame – A single message on a CAN bus, defined by ID, DLC, payload, timestamp, and flags.

    Charts panel – Visualization area plotting decoded signals over time.

    Command Palette – Global search interface for commands (Ctrl/Cmd + P).

    CSV – Comma-separated values text format for logs or exports.

    D¶

    Data Injection – Feature that transmits CAN frames from TrueFidelity to simulate or modify traffic.

    DBC – CAN database file defining messages, signals, scaling, and metadata.

    Derived Signal – A calculated signal based on other signals or raw data via custom formulas.

    E¶

    Export Queue – Panel managing pending export tasks (segments, reports, etc.).

    Export Segment – Export of a selected timeline range to CSV/MDF.

    F¶

    Filter-to-Range – Restricts visible data to the currently selected timeline segment.

    Floating Panel – Panel detached from the main window, useful on multi-monitor setups.

    G¶

    Glossary – This page, containing definitions of commonly used terms.

    H¶

    Heartbeat – Periodic message ensuring systems remain synchronized; often simulated during injection.

    I¶

    Injection Profile – Saved configuration (.tfinject) containing message templates and schedules.

    Interval (Injection) – Schedule sending messages at a fixed cadence.

    L¶

    Layout – Arrangement of panels saved within a project; interchangeable via the layout switcher.

    Loop Mode – Playback mode that repeats the selected timeline range.

    M¶

    MDF4 – Measurement Data Format 4, binary log format optimized for large datasets.

    Messages panel – Displays raw CAN frames with filtering and export capabilities.

    N¶

    Normalization – Process of aligning timestamps (absolute vs. relative) during import.

    P¶

    Payload – Raw byte content of a CAN frame.

    Pinned Signal – Signal saved to the watch list for quick access.

    Playback Buffer – Number of frames preloaded for smooth playback.

    Profile (Layout) – Saved workspace arrangement or injection configuration.

    R¶

    Ramp (Injection) – Schedule that gradually changes message rate over time.

    Report Builder – Tool generating PDF/HTML summaries including annotations, charts, and metrics.

    S¶

    Segment – Time selection on the timeline used for playback looping or export.

    Signals panel – Interface for searching, pinning, and managing decoded signals.

    Snapshot – Screenshot of charts or panels stored in the export folder.

    T¶

    Timeline – Topmost bar showing playback progress, selections, and markers.

    Transport Controls – Play, pause, step, and loop buttons located in the footer.

    TrueFidelity Project (.tfproj) – File storing project configuration, layout references, filters, and metadata.

    U¶

    UI Scaling – Setting controlling interface size for accessibility.

    V¶

    Verbose Logging – Enhanced logging mode for troubleshooting.

    W¶

    Watch List – Collection of pinned signals displayed in the sidebar.

    Workspace – The overall UI environment containing panels, timeline, and controls.

    Z¶

    Zero-copy Import – Links files without copying, preserving disk space (default behavior when importing logs).
