    Troubleshooting¶

    Use this guide to resolve common TrueFidelity issues. If problems persist, gather logs and contact support.

    Before You Begin¶

    1. Confirm you’re running the latest approved version. Check Help → About.
    2. Reproduce the issue and note the exact steps, timestamps, and files involved.
    3. Open Help → Show Logs; keep the folder ready for support if needed.

    Installation & Launch¶

    Installer Fails or Freezes¶

    • Run the installer as administrator (Windows) or using sudo (Linux).
    • Disable antivirus temporarily if it quarantines installer components.
    • Verify disk space and permissions in the target directory.

    Application Won’t Start¶

    • Launch from a terminal to view error output (truefidelity --verbose).
    • Update graphics drivers; try launching with --disable-gpu or --safe-mode.
    • Reset settings by deleting the configuration file under your user profile (document before removal).

    Playback & Timeline¶

    Playback Stutters¶

    • Reduce playback speed or limit visible panels.
    • Move logs to faster storage (SSD/NVMe).
    • Enable Incremental decoding in settings.
    • Check CPU/GPU usage for bottlenecks.

    Timeline Jumps Erratically¶

    • Normalize timestamps when importing logs.
    • Confirm logs are not using different time bases; apply offsets if necessary.
    • Disable conflicting filters that might skip large sections.

    Annotations Missing After Restart¶

    • Ensure autosave is enabled or manually save before closing.
    • Check project folder permissions; if read-only, TrueFidelity can’t save updates.

    Charts & Signals¶

    Charts Blank or Frozen¶

    • Reset layout to reinitialize panels.
    • Remove overloaded signal sets and re-add critical ones.
    • Update GPU drivers; disable hardware acceleration if necessary.

    Signal Not Decoding¶

    • Verify the correct DBC is imported and prioritized.
    • Ensure the message uses the expected ID (standard vs. extended).
    • Check for conflicting definitions; disable duplicates temporarily.

    System Design & ECU Management¶

    ECU Won't Start¶

    • Verify Docker Desktop is running and accessible.
    • Check Console panel for error messages from the container.
    • Ensure binary path is correct and file exists (if specified).
    • Verify system has available resources (CPU, memory, Docker limits).
    • Try starting ECUs one at a time to isolate issues.

    System Canvas Won't Save¶

    • Check file permissions on the save location.
    • Verify no other instance has the system file open.
    • Ensure sufficient disk space.
    • Try saving to a different location as a test.

    Network Connections Not Working¶

    • Verify network edges are properly connected in System Canvas.
    • Check network properties (baud rate, channel assignments).
    • Ensure ECUs are running before expecting traffic.
    • Review Network Traffic panel for actual message flow.

    Build Failures¶

    • Check Build Console panel for compilation errors.
    • Verify build target is correct for your hardware.
    • Ensure toolchain is properly installed and configured.
    • Review ECU properties for correct source paths and compiler options.

    ECU Monitoring¶

    ECU Status Shows "Error" or Won't Start¶

    • Open Console panel and filter to that ECU.
    • Look for error messages indicating the cause (missing binary, port conflicts, resource limits).
    • Check Docker logs: docker ps -a and docker logs <container-id>.
    • Verify container resources aren't exhausted (memory, CPU limits).

    Console Logs Not Appearing¶

    • Verify ECU is running (green status in ECU Status panel).
    • Check Console panel filters aren't hiding all messages.
    • Ensure log level toggles include the message types you expect.
    • Try refreshing the Console panel or reconnecting.

    Resource Monitor Shows Zero or No Data¶

    • Ensure ECUs are actually running (not just defined).
    • Check time range includes recent activity (adjust dropdown).
    • Verify Resource Monitor panel is visible and not minimized.
    • Restart ECUs to reset metrics collection.

    Terminal Connection Fails¶

    • Verify ECU is running (green status).
    • Check Console logs for terminal service errors.
    • Some ECU images don't have terminal support—verify image capabilities.
    • Try reconnecting using the reconnect button in Terminal panel.
    • Check Docker networking: ensure sidecar containers are running.

    Network Traffic Empty¶

    • Ensure ECUs are running and connected via network edges.
    • Check Network Traffic panel filters aren't hiding all messages.
    • Verify ECU firmware is actually sending CAN messages.
    • Check that network type (CAN/Ethernet) matches configuration.
    • Review Console logs for network interface errors.

    Process Monitor Shows No Processes¶

    • Ensure correct ECU is selected.
    • Verify ECU has a running operating system (some minimal images may not support process monitoring).
    • Check that Process Monitor service is enabled for that ECU.

    Data Injection¶

    Injection Not Starting¶

    • Confirm license includes injection feature.
    • Ensure hardware is connected and not in monitor-only mode.
    • Review injection profile for invalid IDs or payload formatting.

    Injection Stops with Errors¶

    • Inspect the injection log for bus-off or permission errors.
    • Reduce frequency; high rates may saturate the bus.
    • Restart the CAN interface or power-cycle hardware.

    Exports & Reports¶

    Export Queue Stuck¶

    • Verify destination drive has free space and write permissions.
    • Cancel the offending task and retry.
    • Check logs for file path errors.

    Report Missing Images¶

    • Ensure screenshots referenced in the report remain in the project folder.
    • Rebuild the report after adding missing assets.

    UI & Panels¶

    Panel Missing¶

    • Use View → Reset Layout.
    • If using multiple monitors, ensure the OS still detects all displays.

    Shortcuts Not Working¶

    • Check for conflicts in Settings → Shortcuts.
    • On macOS, ensure the key combination isn’t reserved by the OS.

    Performance Degradation Over Time¶

    • Clear the Messages panel filters and reduce chart complexity.
    • Restart TrueFidelity to release cached resources.
    • Update to the latest patch that may include performance fixes.

    Logging & Diagnostics¶

    • Enable Verbose logging in Settings → Diagnostics for extra detail.
    • Use Help → Copy Support Bundle if available to gather logs automatically.
    • Include system information (OS version, CPU, RAM, GPU) when reporting issues.

    Contacting Support¶

    1. Prepare the following:
    2. Brief issue description and steps to reproduce
    3. Screenshots or screen recordings
    4. Project file (.tfproj) or sanitized sample log if possible
    5. Log bundle ZIP
    6. Submit through your official support portal or email.
    7. Reference any related ticket numbers or change requests.

    Quick Reference Table¶

    Symptom Quick Fix
    App launches but nothing renders Run with --disable-gpu, update drivers.
    ECU won't start Check Docker is running, review Console logs, verify binary path.
    Terminal connection fails Verify ECU is running, check Console for errors, try reconnect.
    Resource Monitor shows zero Ensure ECUs running, adjust time range, refresh panel.
    Network traffic empty Verify network edges connected, check filters, ensure ECUs sending messages.
    Signals missing Re-import DBC, check priority order.
    Export empty Verify timeline range and filters.
    Crash on exit Delete temp files from %AppData%/TrueFidelity/cache (Windows) or ~/Library/Application Support/TrueFidelity/cache (macOS).
