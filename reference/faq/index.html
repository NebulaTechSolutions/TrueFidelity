    Frequently Asked Questions¶

    Find answers to common user questions about TrueFidelity.

    General¶

    Q: Which operating systems are supported?
    A: Windows 10/11 (x64), macOS 12+, and Linux (x86_64). Use the latest patches for best results.

    Q: Does TrueFidelity require administrative privileges?
    A: Installation may require elevated permissions; day-to-day usage does not unless you access protected CAN interfaces.

    Q: Where are configuration files stored?
    A: Typically under the user profile: %APPDATA%\TrueFidelity (Windows), ~/Library/Application Support/TrueFidelity (macOS), ~/.config/truefidelity (Linux).

    Projects & Files¶

    Q: How do I share a project with colleagues?
    A: Zip the .tfproj file along with referenced logs, definitions, screenshots, and exports. Use relative paths to keep links intact.

    Q: Can I open multiple projects at once?
    A: TrueFidelity opens one project per window. You can run multiple instances if your system resources allow it.

    Q: How do I recover a project after a crash?
    A: Reopen TrueFidelity; autosave restores the last checkpoint. If autosave was disabled, manually load the most recent .tfproj backup.

    Playback¶

    Q: Why does playback stutter on large logs?
    A: Ensure logs reside on an SSD, close unused panels, and reduce chart complexity. See Performance Tips for more suggestions.

    Q: Can I sync playback across multiple machines?
    A: Not directly. Export segments or share projects so teammates can run the same playback on their machines.

    Signals & Definitions¶

    Q: What happens if multiple DBC files define the same signal?
    A: TrueFidelity uses the highest-priority definition. Reorder or disable definitions in the sidebar to select the correct one.

    Q: How do I handle encrypted payloads?
    A: Decrypt them using your organization’s tooling before importing; TrueFidelity doesn’t perform decryption.

    Data Injection¶

    Q: Is injection available in all editions?
    A: Injection requires a license tier that enables it. Check your activation status or contact your admin.

    Q: Can I run injection and playback simultaneously?
    A: Yes. Use separate channels or annotate injected frames to keep them organized.

    Q: How do I stop injection quickly?
    A: Click Stop in the injection panel or use the emergency hardware cutoff on your test bench.

    Exports & Reports¶

    Q: Why is my export empty?
    A: Ensure the selected timeline range contains data and filters aren’t excluding everything.

    Q: Can I customize report templates?
    A: Yes. Clone a template in the Report Builder and adjust branding, sections, and default text.

    Q: Where are exports saved?
    A: By default, in the export folder set under Settings → Exports. The status bar displays the path after each export.

    Performance & Stability¶

    Q: How do I speed up large exports?
    A: Use binary formats (MDF4), run exports from fast storage, and queue them during off-hours.

    Q: The app feels sluggish after hours of use.
    A: Restart TrueFidelity to release cached resources and review Performance Tips for proactive tuning.

    Support¶

    Q: How do I capture logs for support?
    A: Use Help → Show Logs, compress the folder, and include it with your support request.

    Q: What information should I provide to support?
    A: Include OS version, app version, reproduction steps, logs, project or sample logs (if possible), and relevant screenshots.

    Updates & Licensing¶

    Q: How do I update TrueFidelity?
    A: Download the latest installer from your distribution portal and run it. Your settings and projects remain intact.

    Q: Will updates modify my project files?
    A: Projects are forward compatible; however, always keep backups before upgrading.

    Q: How can I transfer a license to a new machine?
    A: Deactivate on the old machine (Help → License) and activate on the new one using your license credentials.

    System Design & ECU Management¶

    Q: How do I add an ECU to my system?
    A: Drag an ECU component from the Toolbox onto the System Canvas. Then configure its properties (binary path, network interfaces) in the Properties panel.

    Q: Can I run multiple ECU systems simultaneously?
    A: Yes, but resource usage scales with the number of ECUs. Each ECU container consumes CPU and memory. Monitor resources using the Resource Monitor panel.

    Q: How do I connect ECUs on different networks?
    A: Use a Gateway ECU component in the Toolbox. Connect it to ECUs on different network segments. Configure network edges (CAN channels) in the Properties panel.

    Q: What happens if an ECU crashes or stops?
    A: The ECU Status panel shows the error state (red indicator). Check the Console panel for error messages. You can restart the ECU from the context menu.

    Q: Can I modify ECU configuration while it's running?
    A: Some changes require stopping the ECU first (e.g., network configuration). Others may take effect on restart. Stop the ECU, modify properties, save the system, then restart.

    ECU Monitoring & Terminal Access¶

    Q: Why can't I connect to an ECU terminal?
    A: Verify the ECU is running (green status). Check Console logs for terminal service errors. Some ECU images may not have terminal support enabled. Ensure appropriate permissions.

    Q: How do I filter logs from a specific ECU?
    A: In the Console panel, type the ECU name in the filter bar. You can also use log level toggles to show only errors, warnings, etc.

    Q: Can I export monitoring data?
    A: Yes. Export Console logs as text/CSV. Export Resource Monitor metrics as CSV or PNG. Export Network Traffic as BLF, MDF4, or CSV.

    Q: Why are my resource graphs showing zero?
    A: Ensure ECUs are running. Check that the Resource Monitor panel is visible and time range includes recent activity. Refresh the panel if needed.

    Q: How do I access terminal for multiple ECUs?
    A: The Terminal panel has one tab per ECU. Click different tabs to switch between ECU terminals. Each tab maintains its own connection and command history.

    Q: Can I run automated commands in terminals?
    A: Terminal access is interactive. For automation, consider using the CLI tools or scripting against the TrueFidelity APIs (if available in your edition).

    Miscellaneous¶

    Q: Can I integrate scripts or automation?
    A: Some editions include CLI utilities. Consult your release notes or support representative for availability.

    Q: Does TrueFidelity support other bus types (LIN, FlexRay)?
    A: Support varies by release. Check the release notes or contact support for the current bus compatibility list.

    Q: How do I change the Catppuccin theme colors?
    A: Edit docs/styles/extra.css or use Settings → Theme if exposed in your build.
